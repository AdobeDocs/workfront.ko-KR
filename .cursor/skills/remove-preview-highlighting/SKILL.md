# 미리 보기 강조 표시 제거(Workfront)

## 범위

**모두**&#x200B;이(가) true인 경우에만 적용:

1. 사용자가 이 워크플로우를 호출했습니다(예: **&quot;미리 보기 강조 표시 제거&quot;** 또는 명확히 동일한 의도).
2. Markdown 파일의 경로에 **이(가) 포함되어 있지**&#x200B;않습니다&#x200B;**`product-announcements`**(릴리스 정보, 베타, `help/quicksilver/product-announcements/` 아래의 공지 등 전체 폴더 트리 제외).
3. Markdown 파일이 아래 **제외된 경로**&#x200B;에 **[나열되지 않음](#excluded-paths)**&#x200B;입니다.
4. Markdown 파일의 앞줄에 **`Courtney`** 줄의 `author:`이(가) 포함되어 있습니다(단독 작성자 또는 공동 작성자).
5. 문서에 **하나 이상의**&#x200B;이(가) 있습니다.
   - 본문 산문 또는 실제 코드 조각의 미리 보기 환경 **언어**(일반적인 패턴: &quot;강조 표시된 정보&quot;, &quot;미리 보기 환경&quot;, &quot;아직 일반적으로 제공되지 않음&quot;, 빠른 릴리스 정보)—**없음** 목차/색인 페이지의 **텍스트만 연결**&#x200B;과(와) 일치(아래 참조) 또는
   - **`class="preview"`**&#x200B;이(가) 있는 모든 HTML 요소(예: `<span class="preview">`, `<div class="preview">`) 또는
   - 미리 보기 코드 조각 **변수**(예: **`{{highlighted-preview}}`** 또는 **`{{highlighted-preview-article-level}}`**).

범위가 명확하지 않은 경우 편집하기 전에 확인하십시오.

**TOC/인덱스 페이지 — 항상 다음 경우에 건너뜁니다.** **항상** **전용** 미리 보기 관련 단어가 **내부** 문서인 Markdown 링크의 표시 텍스트(예: **다른** 문서(예: *미리 보기 샌드박스 환경에서 보고서 보내기*) **및** 파일에 **no** `class="preview"`, **no** 코드 조각 변수 및 **링크 외부 prose**&#x200B;의 **no** 미리 보기 보일러판이 있습니다. 해당 페이지에서 강조 표시를 미리 보는 것은 아닙니다. 이는 TOC 언급일 뿐입니다. 한 파일에만 적용되는 것이 아니라 모든 인덱스/TOC에 적용됩니다.

### 제외된 경로

사용자가 명시적으로 재정의하지 않는 한 인벤토리에 추가하거나 이 워크플로에서 편집하지 마십시오.

- `help/quicksilver/documents/adobe-workfront-for-experience-manager-assets-essentials/send-to-aem.md` — 병렬 미리 보기와 프로덕션 제한 간에 자동화 외부에서 의도한 편집 결정이 필요합니다.
- `help/quicksilver/reports-and-dashboards/reports/creating-and-managing-reports/create-manage-reports.md` — TOC를 보고합니다. &quot;미리 보기&quot; 유일한 신호는 Preview 샌드박스 게재 문서에 대한 링크입니다.

## 필수 워크플로우(루프의 사람)

승인 없이 리포지토리를 **not** 일괄 편집하지 마십시오.

1. **인벤토리**\
   위의 범위 규칙을 충족하는 정렬된 경로 목록을 빌드합니다(저장소를 검색하고 `help/` 트리를 선호함). **생략** **`product-announcements`**&#x200B;에 있는 모든 경로, **[제외된 경로](#excluded-paths)**&#x200B;에 있는 모든 경로 및 **TOC/인덱스 페이지**&#x200B;와(과) 일치하는 **TOC/인덱스** 페이지를 범위 아래에 추가합니다. 나열된 파일에 미리보기 강조 표시가 없다고 하면 실행에서 제거하고 강제로 편집하지 않고 기준을 강화합니다.

2. **시작**\
   목록의 **first** 문서(또는 사용자 이름 경로)로 시작할지 여부를 묻습니다.

3. 문서당 **먼저 표시, 확인 후 편집**
   - 파일을 읽습니다.
   - 편집 내용을 명확하게 제안합니다. **발췌 전/후** 또는 변경 내용에 대한 집중된 차이점 스타일 설명.
   - 파일을 쓰기 전에 명시적 승인(예: &quot;확인&quot;, &quot;적용&quot;, &quot;예&quot;)을 위해 **대기**.

4. **각 파일 뒤**\
   **next** 문서로 이동할지(또는 중지/건너뛰기) 여부를 묻습니다.

## 콘텐츠 규칙(GA: 콘텐츠 미리보기 가 설명서가 됨)

**일반 가용성**&#x200B;에 대한 미리 보기 강조 표시를 제거할 때 목표는 **미리 보기에 대해 문서화된 동작 유지**, **사용되지 않는 &quot;프로덕션&quot;/이전 프로세스** 분기를 삭제한 다음 **미리 보기 전용 레이블 및 래퍼를 제거**&#x200B;하여 주제가 모든 고객의 현재 항목으로 읽히도록 하는 것입니다.

### 병렬 단계

- 문서에 단계(또는 번호가 매겨진 항목)가 **프로덕션에서**(또는 동등: 현재 프로덕션/기존 프로덕션 동작)로 **있고** **parallel** 단계가 **&quot;미리 보기&quot;**(또는 미리 보기 환경)으로 되어 있는 경우:
   - 프로덕션 내 단계(이전 경로)를 **제거**&#x200B;합니다.
   - **미리 보기 중 단계의** substance **을(를) 유지**&#x200B;하지만 **reword**&#x200B;을(를) 통해 미리 보기별로 **그렇지 않음**(미리 보기에서 &quot;미리 보기 중&quot;, &quot;미리 보기 환경&quot; 등 제거). 목록이 일관되게 유지되도록 번호 매기기를 조정합니다.

구조가 모호하면(명확한 병렬 없음) **중지하고** 두 후보를 모두 표시하십시오. 사용자에게 보관할 블록을 요청하십시오.

### 병렬 섹션

- **섹션**(제목 + 본문)이 **프로덕션 환경에서&quot;에 대한 정보**&#x200B;이고 **병렬 섹션** **미리보기 환경에서&quot;에 대한 정보**&#x200B;인 경우:
   - 프로덕션 환경 섹션(대체된 콘텐츠)을 **제거**&#x200B;합니다.
   - **미리 보기 섹션의 콘텐츠로 바꾸기**&#x200B;한 다음 **제거** 미리 보기 환경 문구와 **`class="preview"`** 래퍼를 사용하여 해당 섹션이 중립(GA 준비)이 되도록 합니다.

### 코드 조각 및 상위 문서 공지

- 기능이 GA되면 **미리 보기 전용 보일러판** 블록(강조 표시된 콘텐츠가 미리 보기 전용, 빠른 릴리스, 샌드박스 등임을 설명하는 범위/div 또는 단락)을 제거합니다.
- 사용자가 다른 알림을 유지하라고 지시하지 않는 한 **only** 목적이 미리 보기 사용 가능한 링크 또는 문장을 제거하십시오.

### HTML `class="preview"`

- **,**&#x200B;내부 콘텐츠 유지&#x200B;**`class="preview"`**(태그 내 Markdown/HTML)를 사용하는 요소의 **태그를 열고 닫는**&#x200B;을(를) 제거합니다.
- **`<span class="preview">`**&#x200B;과(와) **`<div class="preview">`**&#x200B;을(를) 모두 처리하고, 다른 태그(예: **`<p class="preview">`**, **`<li class="preview">`**)가 **보이는**(주석 처리 아님) 본문 컨텐츠에 나타나면 동일한 패턴을 처리합니다.
- 목록/표 구조를 보존하고, 래핑을 취소한 후 끊어진 목록 또는 공백이 생기지 않도록 수정하십시오.

### 주석 처리된 섹션(HTML 주석)

- 사용자가 명시적으로 **할 내용을 말하지 않는 경우(예: 전체 댓글 삭제, 댓글 내의 미리 보기 클래스 삭제, GA에 대한 댓글 삭제 등)**&#x200B;삭제, 줄바꿈 또는 다른 방식으로 **수정** HTML 댓글 **`<!-- … -->`**&#x200B;을(를) 수행하지 마십시오.****
- 미리 보기 관련 콘텐츠 또는 **`class="preview"`**&#x200B;이(가) 댓글에 **만** 표시되는 경우 문서를 검토할 때 **이를 호출하십시오**: 댓글에 있는 내용을 말하고 **수행할 작업을 묻기**. **기본값: 주석 섹션을 변경하지 않고 둡니다.**

### 하지 말아야 할 일

- **`product-announcements`**&#x200B;의 경로(릴리스 정보 및 관련)에서 이 워크플로우를 실행하지 마십시오. 인벤토리에서 제외해야 합니다.
- 사용자가 명시적으로 경로를 포함하도록 요청하지 않는 한 **[제외된 경로](#excluded-paths)**&#x200B;에 나열된 경로는 인벤토리를 작성하거나 편집하지 마십시오.
- **자동으로**&#x200B;주석 처리된&#x200B;**(**) 블록을 제거하거나 편집하지 마십시오`<!-- … -->`. 위의 **주석 처리된 섹션**&#x200B;을(를) 따르십시오.
- 이 기능 가용성 패턴(예: 관련 없는 컨텍스트에서 **제품 이름**(으)로 [샌드박스 환경 미리 보기](·))가 **not**&#x200B;인 경우 &quot;미리 보기&quot;를 제거하지 마십시오. 확실하지 않은 경우 판단을 사용하여 문의하십시오.
- 사용자가 요청하지 않는 한 `author:` 또는 관련 없는 Frontmatter를 변경하지 마십시오.
- **표시 → 승인** 단계를 건너뛰지 마십시오.

## 편집 내용을 제시하기 전에 품질 검사

- 분리된 중복 머리글이나 끊어진 단계 번호가 없습니다.
- 도입이 GA와 상충되는 **없이**&#x200B;올바르게 읽습니다(제공된 기능에 대해 &quot;미리 보기에서만&quot; 없음).
