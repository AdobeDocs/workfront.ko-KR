# 쓰기 품질 참조

`writing-quality` 스킬에 대한 자세한 규칙입니다. SKILL.md 파일은 가장 일반적인 규칙을 다룹니다. 이 파일은 구두점, 용어, 대문자 예제 및 절차 서식에 대한 전체 참조를 제공합니다.

&#x200B;---

## 구두점

### 마침표

- 항목이 문장을 완료하는 경우 목록에서 마침표를 사용합니다(&quot;X 메뉴에서 다음을 수행할 수 있습니다.&quot;).
- 항목이 전체 문장인 경우 목록에서 마침표 사용
- 마침표는 주 콘텐츠로 서식을 지정할 수 없습니다(올바른: **완료**&#x200B;를 클릭하세요. / 올바르지 않음: **완료**&#x200B;를 클릭합니다.
- 파일 확장명(.pdf)에서 마침표 사용

### 쉼표

- 옥스퍼드 쉼표 항상: &quot;동기, 수단, 기회&quot;
- 소개 구문 뒤에 쉼표가 있습니다. &quot;문서를 열려면 열기를 클릭합니다.&quot;
- 문장 중간값의 다음 연도 쉼표: &quot;2016년 1월 2일, 발행일...&quot;
- 프로시저에서 &quot;then&quot; 앞에 쉼표를 추가합니다. &quot;**설치**&#x200B;를 클릭한 다음 **인터페이스**&#x200B;를 클릭합니다.&quot;
- 월-년만 포함하는 쉼표 없음: &quot;2016년 1월&quot;

### 아포스트로피

- 수축에 사용: 하지 마세요, 그들은, 그것은
- 단일 소유: (Jen의 컴퓨터) 추가
- s: &#39; (The Joneses&#39; 컴퓨터)만 추가
- &quot;it&#39;s&quot;(it is)와 &quot;its&quot;(it에 속함)를 구분합니다.

### 콜론

- 직접 따옴표, 고유 명사 또는 글머리 기호 목록이 뒤따르지 않는 한 콜론 다음의 소문자
- 목록을 소개하는 문장/조각의 끝에 사용
- 시술 제목 후 결장 없음
- 시각적 요소 또는 코드 세그먼트를 도입하는 콜론이 없습니다.

### 세미콜론

- 연결 없이 두 절 연결
- 자체 구두점을 포함하는 구문 구분
- 절차에서 굵은 텍스트 다음에 세미콜론을 굵게 표시하지 않음
- 하이퍼링크에 포함하지 않음

### 따옴표

- &quot;따옴표&quot;가 아닌 &quot;따옴표&quot;로 이를 참조합니다.
- 닫는 따옴표는 쉼표와 마침표를 벗어납니다.
- 물음표 배치는 인용문인지 문장이 문제인지에 따라 달라진다

### 하이픈

- 가능한 경우 하이픈이 필요한 구조 방지
- 명사 앞에 복합 수정자 하이픈 넣기: &quot;읽기 전용 메모리&quot;
- 하나의 수정자가 &quot;복사 방지 디스크&quot;인 경우 하이픈 넣기
- 하나의 수정자가 숫자인 경우 하이픈 넣기: &quot;12포인트 글꼴&quot;
- 키 조합에 하이픈 없음 — + 사용(Ctrl+I)

### Em 대시

는 제한적으로 사용하십시오. 대신 문장을 재구성하는 것을 고려하세요.

### En 대시

- 범위: 2005-2013
- 빼기 기호 또는 음수
- 복합 형용사를 다른 단어와 연결: &quot;대화 상자 유형 옵션&quot;

### 생략 부호

- 생략된 코드 또는 구문
- 대화 상자가 표시됨을 나타내는 UI 표시
- 인용문에서 생략된 텍스트(문장의 끝을 바꿀 경우 마침표 추가)

### 슬래시

- 선택 항목에 슬래시 없음 — &quot;or&quot; 사용(&quot;he&quot; 대신 &quot;he or&quot;)
- 조합에 사용: 온/오프 스위치
- 인터넷 주소 및 소수 자릿수에 사용

| 그룹 관리자 | &quot;**그룹 관리자** 필드를 찾습니다.&quot; | &quot;그룹 관리자에게 문의하십시오.&quot; |

- 리소스 플래너(계획)

## 절차 서식

### 소개

필요한 경우에만 포함합니다. 기본 설정 형식(순서):

1. 부정사 구문 + 콜론: &quot;시스템의 사용자에 대한 임시 암호를 만들려면:&quot;
2. 문장 + 마침표: &quot;시스템의 사용자에 대한 임시 암호를 만들어야 합니다.&quot;
3. Infinidient 구문 + &quot;다음 단계를 따르십시오.&quot; + 콜론: &quot;사용자를 위한 임시 암호를 생성하려면 다음 단계를 따르십시오.&quot;

### 단계 규칙

- 모든 프로시저에 번호 매기기 목록 사용(한 단계만 사용)
- 단계 내에서 굵게 클릭 가능한 작업 및 UI 요소
- 특정 작업 동사 사용: &quot;type&quot;, &quot;select&quot;, &quot;click&quot; — &quot;set&quot; 또는 &quot;specify&quot; 방지
- 시스템 응답이 발생하기 전이나 그 직후에 설명

시스템 응답 배치의 예:

```
1. Click **Sharing**.
2. In the **Task Access** dialog box, enter the name of the user you want to give task access to.
```

### 조건부 및 선택적 단계

```
3. (Conditional) If you are a member of more than one team, select...
3. (Optional) To add an emoji, click...
3. (Conditional and optional) If you want to add a note...
```

### 하위 단계 및 필드 목록

필수 명령(구성, 지정, 실행)을 사용한 다음 필드를 나열합니다.

```
1. Specify the following:
   a. **Favorite color**: Select your favorite color from the drop-down menu.
   b. **Favorite song**: Type your favorite song.
```

### 여러 메서드

가장 간단한 방법을 먼저 문서화합니다.

```
1. In the **Main Menu**, click **Home**.

   Or

   Click the **Home** icon in the upper-left corner.
```

### 직각 대괄호

- 메뉴/탭 탐색 표시: &quot;**전자 메일** > **알림**&quot;
- 대괄호가 아닌 굵게 표시된 단추 이름
- 대괄호 주변 공백
- 메인 메뉴에 사용하지 않음 — 메인 메뉴 코드 조각 사용

## 참고, 팁, 경고 — 자세한 지침

### 각 항목 사용 시기

**참고**(`>[!NOTE]`):
- 단락 컨텍스트에 맞지 않는 정보
- 사용자의 불만 또는 시간 낭비를 방지하십시오(&quot;이 작업을 완료하려면 팀 라이선스가 있어야 합니다.&quot;).
- 버전 관리 또는 호환성 정보(&quot;이 기능은 레거시 증명 뷰어에서만 사용할 수 있습니다.&quot;)

**팁**(`>[!TIP]`):
- 사용자의 생활을 편리하게 하지만 중요하지 않은 제안
- 예: &quot;Shift 키를 누른 상태에서 여러 문서를 동시에 공유할 수 있습니다.&quot;

**중요**(`>[!IMPORTANT]`):
- 단계 또는 절차에 대한 중요한 정보

**경고**(`>[!WARNING]`):
- 사용자에게 잠재적인 데이터 손실에 대해 경고

### 규칙

- 섹션당 최대 1개
- 여러 개의 메모/팁/경고를 스택하지 마십시오.
- 글머리 기호 목록과 함께 관련 메모를 하나로 결합
- 새로운 기능을 발표하는 데 노트를 사용하지 마십시오. 문서 텍스트를 다시 작성하십시오.

## DQTI 품질 특성

### 사용하기 쉬움

| 특성 | 주요 질문 |
|---------------|---------------|
| 작업 방향 | 이는 사용자가 수행할 수 있는 작업에 중점을 둡니다. 머리글은 명령동사인가? |
| 정확도 | 용어가 UI와 정확히 일치합니까? 기술 세부 사항이 정확합니까? |
| 완성도 | 행동하기에 충분한 정보가 있습니까? 제거할 불필요한 정보가 있습니까? |

| 특성 | 주요 질문 |
|---------------|---------------|

| 구체성 | 구체적인 예가 있습니까? UI 요소의 이름이 정확하게 지정되었습니까? |
| 스타일 | 능동태? 현재 시제? 일관된 용어? |

### 찾기 쉬움

|---------------|---------------|

| 시각 효과 | 3개 이상의 항목을 나열하시겠습니까? 비교를 위한 표 텍스트 문단 없나? |

## 일반 수정 사항

| 문제 | 수정 |
|-------|-----|
| &quot;소개하게 되어 기쁩니다...&quot; | 마케팅 언어 제거 — 기능을 직접 명시합니다. |
| &quot;X 구성이 업데이트되었습니다.&quot; | &quot;이제 X를 구성할 수 있습니다.&quot;(활성, 작업 중심) |
| &quot;권장되는 작업...&quot; | &quot;다음을 권장합니다...&quot; |
| &quot;사용자는 이 기능을 활용하여 다음을 수행할 수 있습니다.&quot; | &quot;이 기능을 사용하여 다음을 수행할 수 있습니다.&quot; |
| &quot;값 설정&quot; / &quot;옵션 지정&quot; | &quot;값 입력&quot; / &quot;옵션 선택&quot; |
| &quot;비활성화됨&quot; / &quot;회색으로 표시됨&quot; | &quot;사용할 수 없음&quot; |
| &quot;dropdown&quot;(독립 실행형) | &quot;menu&quot; 또는 &quot;drop-down menu&quot; |
| &quot;그/그녀&quot; | &quot;they&quot; 또는 재구성 |
| 누적 메모 | 글머리 기호와 하나의 메모로 결합 |
| 굵게 외부 절차 단계 | 굵게 제거 |
