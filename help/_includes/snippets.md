## 연결된 폴더 설정 {#setup-linked-folder}

Workfront 프로젝트에서 Experience Manager에 연결된 폴더를 만들 수 있도록 허용할 수 있습니다. 폴더가 연결되면 폴더에 추가된 모든 자산이 Workfront 및 Experience Manager 모두에 자동으로 표시됩니다. 자산이 Workfront의 연결된 폴더에 처음 추가되면 자산의 메타데이터가 Experience Manager Assets에 푸시됩니다.

아래 단계에서는 연결된 폴더를 만들 위치를 나타냅니다. 각 통합에는 연결된 모든 폴더에 대해 하나의 위치만 있을 수 있습니다.

연결된 폴더를 설정하려면 다음을 수행하십시오.

1. 전환 **연결된 폴더를 사용하도록 설정했습니다.** 설정
1. 이 통합과 관련된 모든 링크된 폴더를 라이브로 지정할 폴더 경로를 선택합니다.
   ![연결된 폴더 설정 화면](assets/linked-folder-setup.png)

   >[!NOTE]
   >
   >연결된 폴더를 만들려면 Adobe Experience Manager Assets에서 지정된 폴더에 대한 쓰기 액세스 권한이 있어야 합니다.

1. **저장**&#x200B;을 클릭합니다.

## 하나 이상의 상태에 대한 완료 단추 {#configure-done-button}

완료 단추를 사용하여 작업 항목에 적용되는 상태를 변경할 수 있습니다. 여러 상태를 설정하고 사용자가 적절한 상태를 선택할 수도 있습니다.

1. 을(를) 클릭합니다. **[!UICONTROL 기본 메뉴]** 아이콘 ![기본 메뉴](assets/main-menu-icon.png) Adobe Workfront의 오른쪽 위 모서리에서 (사용 가능한 경우) **[!UICONTROL 기본 메뉴]** 아이콘 ![기본 메뉴](assets/main-menu-icon-left-nav.png) 왼쪽 위 모서리에서 **[!UICONTROL 팀]**.

1. 을(를) 클릭합니다. **[!UICONTROL 팀 전환]** 아이콘을 클릭한 다음 드롭다운 메뉴에서 새 팀을 선택하거나 검색 막대에서 팀을 검색합니다.
1. 을(를) 클릭합니다. **[!UICONTROL 자세히]** 메뉴를 클릭한 다음 **[!UICONTROL 편집]**.
1. 를 찾습니다. **[!UICONTROL 완료 단추]** 섹션 아래쪽에서 **[!UICONTROL 팀 설정]** 페이지.

1. 각 작업 항목 유형에 대해 상태를 하나 이상 선택합니다.

   >[!NOTE]
   >
   >작업 또는 문제에 대한 상태를 선택할 때는 다음 사항을 고려하십시오.
   >
   >* 각 작업 항목 유형에 대해 하나의 상태를 선택하면 사용자가 해당 항목에 대해 완료 를 클릭하면 작업이나 문제 상태가 해당 상태로 설정됩니다. 각 작업 항목 유형에 대해 여러 상태를 설정하면 완료 단추에 드롭다운 메뉴가 추가되고 작업 항목의 상태를 변경하려면 사용자가 상태를 선택해야 합니다.
   >* 완료 단추에는 시스템 수준 상태만 연결할 수 있습니다. 그룹별 상태를 작업 항목 상태와 연결할 수 없습니다.
   >* 항목에 지정된 사용자가 완료 버튼과 연관된 상태에 항목을 놓으면, 선택된 상태가 완료 또는 마감 상태인지 또는 작업 상태인지에 관계없이 해당 사용자에 대해 완료된 것으로 표시됩니다.

   >   
   >   
   >  예를 들어, 완료 단추를 진행 중 단추와 연결하면 작업 항목이 새로 작성에서 진행 중으로 변경하는 사용자에게 완료 상태로 표시됩니다.
   >   
   >* 문제 유형은 사용자 지정할 수 있으며 환경에 아래 나열된 이름과 다른 이름을 가질 수 있습니다.\
      >  다음은 기본 작업 및 문제 유형입니다.
      >     
      >   * 작업
      >   * 문제
      >   * 요청
      >   * 순서 변경
      >   * 버그 신고


   작업이나 문제가 여러 사용자에게 할당된 경우 드롭다운 메뉴에 팀에 대해 선택한 여러 상태 외에 &quot;내 부품으로 완료&quot; 옵션이 표시됩니다.

1. 클릭 **[!UICONTROL 변경 내용 저장]**.

## 홈 팀과 사용자 연결 {#associate-users-with-home-team}

완료 단추 기능을 사용자가 볼 수 있도록 변경하려면 설정을 변경한 팀을 사용자의 홈 팀으로 만들 수 있습니다.

사용자를 홈 팀과 연결하려면
