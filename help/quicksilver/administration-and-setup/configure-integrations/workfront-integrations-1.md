# [!DNL Adobe Workfront] 통합

Adobe Workfront을 타사 애플리케이션 및 기타 애플리케이션과 통합할 수 있습니다 [!DNL Adobe] 제품. 통합은 Workfront의 유틸리티를 확장하고 조직의 요구 사항에 맞게 조정할 수 있습니다.

통합 [!DNL Workfront] 다음 카테고리에 속합니다.

## 내장(기본) 통합

[!DNL Workfront] 에서는 Workfront 애플리케이션에서 직접 또는 를 설치하여 다른 애플리케이션에서 직접 구성할 수 있는 다양한 통합을 제공합니다 [!DNL Workfront] 해당 애플리케이션에 대한 추가 기능입니다.

이러한 통합 중 일부는 추가 비용 없이 사용할 수 있지만, 다른 통합에는 추가 구매가 필요합니다.

* [기본 제공 통합 시작](#get-started-with-built-in-native-integrations)
* [추가 비용 없이 통합 사용 가능](#integrations-available-at-no-additional-cost)
* [추가 구매를 필요로 하는 통합](#integrations-requiring-an-additional-purchase)

### 기본 제공 통합 시작 {#get-started-with-built-in-native-integrations}

기본 제공(기본) 통합을 시작하려면 다음 워크플로우를 따르십시오.

1. **요구 사항에 가장 적합한 기본 통합을 결정합니다**

   통합을 선택할 때는 사람들이 이미 정기적으로 사용하는 애플리케이션을 선택하는 것이 중요합니다. 통합 [!DNL Workfront] 자주 또는 정기적으로 사용되는 애플리케이션을 사용하면 사용자 간 채택률이 높아집니다.

   사용 가능한 통합 목록에 대해서는 다음을 참조하십시오 [추가 비용 없이 통합 사용 가능](#integrations-available-at-no-additional-cost) 및 [추가 구매를 필요로 하는 통합](#integrations-requiring-an-additional-purchase).

1. **기술 관리자 식별**

   통합하려는 애플리케이션에 대한 조직의 기술 관리자 식별 [!DNL Workfront]. 예를 들어 [!DNL Jira], 회사의 [!DNL Jira] 관리 및 플랫폼 내의 모든 구성 요구 사항에 참여합니다. 또한 이 관리자가 [!DNL Workfront] 고객 지원.

   관리자를 시작하려면 해당 통합에 대한 특정 지침을 제공해야 합니다. 이러한 지침에 대한 링크는 [추가 비용 없이 통합 사용 가능](#integrations-available-at-no-additional-cost).

1. **참여 [!DNL Workfront] 고객 지원**

   관리자에게 기본 통합 설정과 관련된 문제가 있는 경우, [!DNL Workfront] 고객 지원 을 참조하십시오. While [!DNL Workfront] 는 통합을 설정하지 않을 것이며, Adobe는 사용자가 직접 이 통합을 수행하는 데 필요한 모든 필수 도구와 리소스를 제공하기 위해 노력하고 있습니다. 고객 지원에 직접 문의하거나 광범위한 리소스 문서를 통해 통합을 쉽게 설정할 수 있습니다.

   고객 지원에 연결하려면 [고객 지원에 문의](../../workfront-basics/tips-tricks-and-troubleshooting/contact-customer-support.md).

   도움말 문서에 대한 링크는 [추가 비용 없이 통합 사용 가능](#integrations-available-at-no-additional-cost). 및 [추가 구매를 필요로 하는 통합](#integrations-requiring-an-additional-purchase).

   지원 사례 만들기에 대한 지침은 [고객 지원에 문의](../../workfront-basics/tips-tricks-and-troubleshooting/contact-customer-support.md).

   >[!NOTE]
   >
   >지원 사례를 제출하려면 조직의 지정된 지원 담당자여야 합니다. 사례를 제출하면 다른 시스템에 대한 기술 관리자를 대화에 추가할 수 있습니다. 직접 지원에 문의할 수도 없습니다.

사용 가능한 기본 통합 기능이 회사의 모든 요구 사항을 충족하지 않을 수 있음을 이해합니다. 이러한 경우 계정 관리자에게 문의하여 사용자 지정 통합에 대해 논의하십시오.

## 파트너 통합

[!DNL Workfront] 에는 을 보강하는 십여 개의 통합이 포함된 풍부한 파트너 에코시스템이 있습니다 [!DNL Workfront] 경험으로 제어됩니다.

기존 파트너 오퍼링의 예로는 [!DNL Salesforce.com], [!DNL Oracle Responsys], 및 [!DNL Microsoft Visual Studio]. 이러한 통합 및 기타 흥미로운 통합에 대한 자세한 내용은 [[!DNL Workfront] Marketplace](https://www.workfront.com/enterprise/marketplace/apps/).

## [!DNL Workfront] API

다음 [!DNL Workfront] API를 사용하면 [!DNL Workfront] 공용 API를 활용하여 경험을 구축할 수 있습니다. 개방형 API를 사용하여 자체 통합을 만들 수 있습니다. 에서 사용 가능한 객체의 자세한 목록을 보려면 [!DNL Workfront] API인 경우 [!DNL Workfront] [API 탐색기](../../wf-api/general/api-explorer.md).
