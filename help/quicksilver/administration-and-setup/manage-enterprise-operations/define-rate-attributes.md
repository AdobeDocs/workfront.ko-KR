# 비율 속성 정의

비율 속성은 작업 역할 이상의 비율에 차원을 추가할 수 있도록 함으로써 Adobe Workfront의 비율 카드 및 비율 기능을 확장합니다. 이는 직무 역할뿐만 아니라 에이전시, 위치, 브랜드, 비용 센터 등과 같은 요인에 따라 요금이 달라지는 에이전시와 기업에 매우 중요합니다.
이러한 속성을 결합하면 Workfront에서 자동으로 올바른 할당 비율을 선택하여 프로젝트 간 재무 정확성과 일관성을 보장할 수 있습니다.

>[!IMPORTANT]
>
>비율 속성은 일회성 기본 설정입니다.

속성을 활성화하고 등급 카드 및 요율에 적용하고 나면 나중에 변경하면 전체 재무 설정에서 데이터 무결성이 손상될 수 있습니다.

## 비율 속성 개요

비율 속성은 다음과 같은 이유로 일회성 설정으로 간주됩니다.

* 속성이 활성화되면 재무 데이터 모델의 일부가 됩니다.
* 비율, 지정, 계획 값 및 실제 값은 모두 선택한 속성 값에 따라 다릅니다.
* 나중에 속성을 변경(이름 변경, 제거 또는 재정렬)하면 다음과 같은 문제가 발생할 수 있습니다.

   * 비율과 속성 간의 연계 상실
   * 잘못되거나 &quot;고립된&quot; 비율
   * 청구 및 보고의 오정렬

이러한 이유로 속성은 초기 Workfront 구현 중에 신중하게 디자인되고 이후에 변경되지 않습니다.

### 비율 속성으로 사용되는 객체

Workfront은 현재 속도 속성으로 사용할 수 있는 세 가지 시스템 개체를 지원합니다.

* **그룹**: _Agency_ 또는 _비즈니스 단위_(으)로 이름이 바뀌는 경우가 많습니다.
* **회사**: _브랜드_, _클라이언트_ 또는 _고객_&#x200B;을 나타낼 수 있습니다.
* **위치**: 일반적으로 _시장_, _지역_ 또는 _사무실_&#x200B;로 사용됩니다.

  위치는 최대 3개 수준까지 계층적으로 정의됩니다. (예: &quot;위치&quot;를 LA로 정의하는 경우 캘리포니아 및 미국도 비율 일치에 사용됩니다.)

속성을 설정할 때 조직의 용어와 일치하도록 각 객체의 이름을 변경할 수 있습니다.
예:

* &quot;에이전시&quot; 레이블 = 그룹 개체 참조
* &quot;비용 부서&quot; 레이블 = 하위 그룹 객체 참조
* &quot;Location&quot; label = Location 개체 참조

이를 통해 Workfront의 데이터 모델 무결성을 유지하면서 비즈니스 구조를 미러링할 수 있습니다.

### Workfront의 비율 속성에 대한 참고 사항

* Workfront은 최대 5가지 수준의 특성을 지원합니다. 시스템은 항상 속성 계층을 따르며 사용 가능한 가장 구체적인 일치를 선택합니다.

   * 0 = 일반 기본 요금
   * 1 - 5 = 점진적으로 더 구체적인 비율

* 비즈니스(에이전시, 브랜드, 시장, 비용 센터 등)를 반영하도록 속성의 이름을 변경할 수 있습니다.
* 설정은 한 번만 이루어집니다. 나중에 속성을 변경하면 재무 데이터 무결성이 손상될 수 있습니다.
* 시스템의 어느 곳에서도 참조되지 않는 속성은 안전하게 삭제할 수 있습니다.

  그러나 속성이 이미 사용 중인 경우(등급 카드, 사용자 프로필, 리소스 또는 할당에서 참조) 삭제는 데이터 무결성을 보호하기 위해 차단됩니다. 이러한 경우, 특히 API 호출을 통해 속성을 제거하려고 하면 오류가 발생합니다.

* 실행 전 테스트: 파일럿 비율 카드를 만들고 올바른 비율이 할당에서 해결되는지 확인합니다.
* 설정 문서 작성: 팀이 비율이 작동하는 방식을 이해할 수 있도록 비율 속성 설정을 팀과 공유합니다.

### 여기서 비율 속성을 적용할 수 있습니다.

비율 속성은 Workfront에서 비율이 존재하는 모든 영역에서 지원됩니다.

* 비율 카드: 직무 역할과 속성별로 비율을 정의합니다.
* 프로젝트 레벨 대체: 프로젝트 레벨에서 비율을 재정의할 때 속성을 적용합니다.
* 작업 역할(설정): 속성을 사용하여 기본 작업 역할 비율을 설정합니다.
* 사용자(사용자 프로필): 개별 사용자에게 기본 속성을 할당하면 할당이 자동으로 올바른 비율로 확인됩니다.

## 요금 속성 구성

각 속성에는 일반 속성 및 필터를 포함하여 구성 가능한 옵션 세트가 있습니다.
필터는 비율을 정의할 때 속성 값을 제안하고 확인하는 방법을 제어합니다. 속성 선택 사항을 일관되게 유지하고, 사용자에게 유효한 옵션을 안내하며, 잘못된 조합을 방지하는 데 필수적입니다.

{{step-1-to-setup}}

1. 왼쪽 패널에서 **비율 특성**&#x200B;을 클릭합니다.
1. 다이어그램에서 더하기 기호 아이콘을 클릭하여 속성을 추가합니다.

   >[!NOTE]
   >
   >다이어그램에는 최대 5개의 속성을 사용할 수 있습니다. 위에서 아래로의 순서는 속성이 적용되는 방식의 계층을 정의합니다. 다이어그램을 왼쪽에서 오른쪽으로 표시하려면 **회전** 아이콘 ![회전 아이콘](assets/rotate-attribute-view-icon.png)을 클릭합니다. 확대하거나 축소하여 화면에 다이어그램을 맞출 수도 있습니다.

1. 속성을 선택하여 화면 오른쪽에 있는 구성 패널을 엽니다.

   ![속도 특성 구성](assets/configure-rate-attributes.png)

1. 객체(그룹, 회사, 위치)의 이름을 업무에 필요한 조건(예: 에이전시, 위치, 비용 부서)으로 변경합니다.
1. 각 특성에서 **저장**&#x200B;을 클릭하여 명명 규칙을 저장합니다.

   이러한 속성의 이름은 시스템의 모든 요금 카드 및 요금에 표시됩니다.

1. 구성 패널에서 각 속성에 대한 속성을 설정합니다.

   * **필수**: 특성이 요율에 대한 필수 필드인지 여부를 선택합니다.
   * **수익 계산에 사용**: 청구 요금 계산에 이 특성을 포함합니다.
   * **비용 계산에 사용**: 이 특성을 비용 계산에 포함합니다.

     >[!NOTE]
     >
     >재무 계산에서 속성이 작동하려면 계산 옵션 중 하나 이상을 선택해야 합니다.

   * (선택 사항) **계층 구조**: 도시 > 주 > 국가와 같이 특성이 부모-자식 관계를 준수하도록 허용합니다.

     이 속성은 Location 개체에만 사용할 수 있습니다.

### 속성에 대한 필터 정의

속성에는 두 가지 유형의 필터를 사용할 수 있습니다.

* 제안 필터는 시스템 논리나 이전 속성 선택에 따라 사용 가능한 옵션 목록의 범위를 좁힙니다. 드롭다운을 상황에 맞게 변경하고 쉽게 사용할 수 있도록 합니다.

  예: 에이전시 > 위치 > 비용 센터

  이 설정에서 비용 부서 속성에는 에이전시와 위치를 모두 참조하는 제안 필터가 있어야 합니다.

  비율을 추가할 때 먼저 Agency = &quot;Star&quot;를 선택하면 위치 드롭다운에 &quot;Star&quot;에 속하는 위치만 제안됩니다.

  그런 다음 비율로 위치 = 시카고를 선택하면 비용 센터 드롭다운에 &quot;Star&quot;와 시카고에 연결된 비용 센터만 표시됩니다.

* 관계 필터는 속성 간의 종속성 체인을 설정합니다. 시스템이 속성이 서로 어떻게 관련되는지 이해하고 유효한 종속성을 적용하는지 확인합니다.

  예: 에이전시 > 위치 > 비용 센터

  이 설정에서는 기관 속성에 유효한 위치 및 비용 부서와 연결된 관계 필터가 있어야 합니다.

필터는 항상 양방향으로 구성해야 합니다. 속성 A에 속성 B와의 관계 필터가 있는 경우 속성 B에는 속성 A로의 제안 필터가 있어야 합니다. 이를 통해 데이터 무결성과 깨끗한 사용자 경험을 보장합니다.

1. 옵션을 선택하여 구성 패널에서 속성에 대한 제안 필터 및 관계 필터를 정의합니다.

   * **필터 형식**:

      * **Standard** 필터는 특성 개체에 유니버설 조건을 적용합니다. 예를 들어, 위치 > 활성 = True(활성 위치만 표시됨)입니다.

        다른 속성을 선택했는지 여부에 관계없이 항상 표준 필터가 적용됩니다.

      * **특성** 필터는 한 특성을 체인의 다른 특성에 연결합니다. 예를 들어, 위치 > 참조 = 에이전시(선택한 에이전시에 연결된 위치만 표시됨)입니다.

        참조된 속성에 값이 있는 경우에만 속성 필터가 적용됩니다. 예를 들어 기관을 선택하면 유효한 위치만 제안됩니다. 에이전시가 비어 있으면 모든 위치가 표시됩니다(위치에 적용된 표준 필터로 제한할 수 있음).

   * **필드**: 위치 ID 또는 활성 플래그와 같은 특성 개체의 직접 필드입니다.
   * **연산자**: 이러한 옵션은 선택한 필드 유형에 따라 다릅니다. 예를 들면 다음과 같음, 같지 않음, 비어 있음, 참/거짓이 있습니다.
   * (표준 필터 유형만 해당) **값**: 예를 들어, 활성 = True입니다.
   * (특성 필터 형식만 해당) **참조**: 이 필터가 종속되는 특성(예: Agency)입니다.
   * (특성 필터 유형만 해당) **참조 필드**: 참조된 특성의 필드로, 일치해야 합니다(예: Agency ID).

1. 각 특성에서 **저장**&#x200B;을 클릭하여 속성 및 필터를 저장합니다.
