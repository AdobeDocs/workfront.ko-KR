# Adobe Workfront의 내장된 액세스 수준

6개의 기본 제공 액세스 레벨은 각각 특정 사용자 유형에 맞게 설계되었습니다.

* 시스템 관리자
* 플래너
* 보조
* 검토자
* 요청자
* 외부 사용자

액세스 수준에 따라 대부분의 Workfront 개체 유형에서 최대 3개의 설정을 사용할 수 있습니다.

<table style="table-layout:auto">
    <tr>
        <td>편집</td>
        <td>사용자는 Workfront 개체를 만들고, 편집하고, 삭제하고, 공유할 수 있습니다</td>
    </tr>
    <tr>
        <td>보기</td>
        <td>사용자는 Workfront 개체를 검토하고 공유할 수 있습니다</td>
    </tr>
    <tr>
        <td>액세스 권한 없음</td>
        <td>사용자는 Workfront 개체에 액세스할 수 없습니다</td>
    </tr>
</table>

사용자 정의 계획자, 작업자, 요청자 또는 검토자 액세스 레벨이 필요한 경우 기본 액세스 레벨을 복사하여 다양한 Workfront 객체 유형에 대해 허용할 액세스 양을 결정할 수 있습니다. 사용자 지정 액세스 수준을 만들거나 기본 제공 액세스 수준 중 하나를 수정하는 방법에 대한 자세한 내용은 [사용자 정의 액세스 수준 만들기 또는 수정](../../../administration-and-setup/add-users/configure-and-grant-access/create-modify-access-levels.md).

>[!IMPORTANT]
>
>사용자를 설정한 후 참조할 수 있도록 기본 액세스 수준을 그대로 두는 것이 좋습니다.

이러한 액세스 수준에 대한 일반적인 정보는 [액세스 수준 개요](../../../administration-and-setup/add-users/access-levels-and-object-permissions/access-levels-overview.md).

## 시스템 관리자 액세스 수준

플랜 라이센스에 첨부된 이 기본 액세스 레벨은 Adobe Workfront 시스템 관리를 담당하는 사용자를 위해 설계되었습니다. 기본 제공 액세스 수준을 수정할 수 없습니다.

시스템 관리자 액세스 수준을 가진 사용자는 Workfront 내에서 모든 작업을 수행할 수 있습니다. 다른 모든 사용자가 Workfront에 입력한 모든 Workfront 개체 및 정보를 보고 편집할 수 있습니다.

또한 전체 설정 영역에 액세스할 수 있으며 시스템 수준에서 모든 설정을 변경할 수 있습니다. 메인 메뉴의 모든 영역에 액세스할 수 있습니다 ![](assets/main-menu-icon.png).

자세한 내용은 [사용자에게 전체 관리자 액세스 권한 부여](../../../administration-and-setup/add-users/configure-and-grant-access/grant-a-user-full-administrative-access.md).

## 계획자 액세스 수준

계획 라이센스에 첨부된 이 액세스 수준은 다음과 같은 용도로 설계되었습니다.

* 그룹, 팀, 프로젝트 및 리소스의 관리자
* 작업, 프로젝트, 포트폴리오 및 프로그램 계획, 생성 및 관리를 담당하는 사람
* 작업(작업 및 문제)을 다른 사용자에게 할당할 책임이 있는 사람
* 보고서를 작성하고 작업표, 작업 항목 및 문서를 승인하는 사용자
* 주 메뉴의 모든 영역에 액세스해야 하는 사용자 ![](assets/main-menu-icon.png)

계획자 기본 제공 액세스 레벨의 사용자 정의 버전을 생성하고 다양한 Workfront 객체 유형에 대해 액세스 권한을 허용하는 양을 결정할 수 있습니다. 자세한 내용은 [Adobe Workfront의 내장된 액세스 수준](#Customiz) 참조하십시오.

다음은 계획자 액세스 레벨에서 객체에 사용할 수 있는 가장 높은 액세스 설정입니다.

| Workfront 개체 유형 | 액세스 권한 없음 | 액세스 권한 보기 | 액세스 편집 |
|---|---|---|---|
| 프로젝트 |   |   | ✓ |
| 작업 |   |   | ✓ |
| 문제 |   |   | ✓ |
| 포트폴리오 |   |   | ✓ |
| 프로그램 |   |   | ✓ |
| 보고서(대시보드 및 달력 보고서 포함) |   |   | ✓ |
| 필터, 보기 및 그룹화 |   |   | ✓ |
| 문서 |   |   | ✓ |
| 사용자 |   |   | ✓ |
| 템플릿 |   |   | ✓ |
| 재무 데이터 |   |   | ✓ |
| 리소스 관리 |   |   | ✓ |
| 시나리오 플래너 |   |   | ✓(기본 설정은 액세스 없음입니다.) |
| Workfront 목표 |   |   | ✓(기본 설정은 액세스 없음입니다.) |

## 작업자 액세스 수준

작업 라이선스에 첨부된 이 기본 액세스 수준은 Workfront에서 작업을 수행할 사용자를 위해 설계되었습니다. 그들은 그 일을 계획하지 않습니다. 그들은 그것을 완성합니다.

이 액세스 수준을 가진 사용자:

* 작업 항목에 할당되며, 여기에서 작업 시간을 기여 및 기록할 수 있습니다
* 작업표와 문서를 승인할 수 있지만 작업표는 승인할 수 없습니다
* 보고서에 액세스하고 공유할 수 있습니다
* 시스템의 다른 사용자와 통신할 수 있습니다
* 주 메뉴의 모든 영역에 액세스할 수 있습니다 ![](assets/main-menu-icon.png)하지만 &quot;사용자&quot; 영역은 Teams로 지정됩니다. 팀 영역에서 이 액세스 수준을 가진 사용자는 해당 팀에 할당된 작업과 함께 자신이 속한 팀만 볼 수 있습니다.
* 프로젝트, 포트폴리오, 프로그램 또는 보고서를 만들 수 없는 개체를 만들 수 있는 기능이 제한됩니다.

Worker 기본 제공 액세스 수준의 사용자 정의 버전을 생성하고 다양한 Workfront 개체 유형에 대해 허용되는 액세스 권한을 결정할 수 있습니다. 자세한 내용은 [Adobe Workfront의 내장된 액세스 수준](#Customiz) 참조하십시오.

다음은 작업자 액세스 수준에서 객체에 사용할 수 있는 가장 높은 액세스 설정입니다.

|---|---|---|---|
| 프로젝트 |   |   | ✓(제한: 사용자는 프로젝트를 공유하고, 작업 및 문제를 만들고, 이미 연결된 사용자 지정 양식에서만 데이터를 편집할 수 있습니다.) |

## 검토자 액세스 수준

검토 라이센스에 첨부된 이 액세스 수준은 다른 사용자로부터 작업을 요청하고 작업을 검토하고 승인하는 경영진에게 적합합니다. 프로젝트 소유자 또는 팀 구성원이 아니지만 작업 항목을 보려면 Workfront에 액세스해야 합니다.

예를 들어, 이 액세스 수준을 가진 이해 관계자가 Workfront에 로그인하여 마케팅 자료를 계속 검토하고, 작업 중인 작업 업데이트를 확인하고, 문서, 승인, 보고서 및 일정을 검토할 수 있습니다.

검토자 액세스 수준을 가진 사용자:

* 작업 항목을 할당하거나 시간표를 승인할 수 없습니다.
* 기본 메뉴에서 요청 및 문서에 액세스할 수 있습니다 ![](assets/main-menu-icon.png).
* 프로젝트, 포트폴리오, 프로그램 또는 보고서를 만들 수 없는 개체를 만들 수 있는 기능이 제한됩니다.

검토자 기본 액세스 수준의 사용자 지정 버전을 만들고 다양한 Workfront 개체 유형에 대해 허용되는 액세스 양을 결정할 수 있습니다. 자세한 내용은 [Adobe Workfront의 내장된 액세스 수준](#Customiz) 참조하십시오.

작업자 액세스 레벨보다 프로젝트 및 작업에 대해 제한적입니다. 다음은 검토자 액세스 레벨의 객체에 사용할 수 있는 가장 높은 액세스 설정입니다.

## 요청자 액세스 수준

요청 라이센스에 첨부된 이 기본 액세스 수준은 Workfront에서 간단한 작업 요청을 만들고 받는 사용자를 위해 설계되었습니다. 기본적으로 요청 영역으로 제한됩니다.

예를 들어 사용자는 조직의 도움말 데스크 요청 큐에 문제를 기록할 수 있습니다.

기본 제공 액세스 수준을 사용하는 사용자:

* 요청을 만들고 해당 요청을 업데이트할 수 있습니다
* 문서 업로드 및 승인 가능
* 제출한 문제의 상태를 검토할 수 있습니다
* 작업 항목에 할당할 수 없습니다.
* 기본 메뉴에서만 요청에 액세스할 수 있습니다 ![](assets/main-menu-icon.png). 요청 큐에 대한 자세한 내용은 [요청 큐 만들기](../../../manage-work/requests/create-and-manage-request-queues/create-request-queue.md).

요청자 기본 제공 액세스 수준의 사용자 지정 버전을 만들고 다양한 Workfront 개체 유형에 대해 허용되는 액세스 양을 결정할 수 있습니다. 자세한 내용은 [Adobe Workfront의 내장된 액세스 수준](#Customiz) 참조하십시오.

## 외부 사용자 액세스 수준

이 액세스 수준은 유료 Workfront 라이선스에 연결되어 있지 않습니다. Workfront에 로그인하지 않지만 때때로 문서를 검토, 다운로드 또는 봐야 하는 외부 컨설턴트와 같은 공동 작업자를 위해 설계되는 가장 제한적인 액세스 수준입니다.

Workfront 사용자는 외부 사용자가 시스템에 로그인할 수 없어도 외부 사용자에게 작업을 할당할 수 있습니다. 그러나 우리는 그 일이 그 제도에서 해결되지 않은 채로 남아있을 것이기 때문에 이에 대해 충고한다.

외부 사용자 액세스 수준을 가진 사용자:

* 공유된 문서 및 달력 보고서만 볼 수 있습니다
* 문서 및 달력 보고서를 공유하는 사용자를 확인합니다
* 공유된 문서를 승인합니다

이 액세스 수준을 수정할 수 없습니다.

>[!IMPORTANT]
>
>외부 사용자는 설정의 시스템 환경 설정 영역에서 &quot;Workfront 계정이 없는 사용자와 이메일 주소를 사용하여 공동 작업&quot; 옵션이 활성화된 경우에만 사용할 수 있습니다. 자세한 내용은 [시스템 보안 환경 설정 구성](/help/quicksilver/administration-and-setup/manage-workfront/security/configure-security-preferences.md).
