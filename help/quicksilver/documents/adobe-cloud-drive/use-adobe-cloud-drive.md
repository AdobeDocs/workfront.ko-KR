# Adobe 클라우드 드라이브 사용

Adobe Cloud Drive를 설치한 후에는 Finder 또는 파일 탐색기에서 직접 Adobe 클라우드 스토리지 프로젝트로 작업할 수 있습니다. 모든 애플리케이션에서 파일을 열고 편집하고, 오프라인으로 작업하고, Adobe Cloud Drive에서 변경 사항을 클라우드에 동기화하도록 할 수 있습니다.

Adobe Cloud Drive 설치에 대한 자세한 내용은 [Adobe Cloud Drive 설치](/help/quicksilver/documents/adobe-cloud-drive/install-adobe-cloud-drive.md)를 참조하세요.

## 프로젝트 액세스

1. Finder(Mac) 또는 파일 탐색기(Windows)를 엽니다.
1. **Adobe 클라우드 드라이브**(으)로 이동합니다.
1. Workfront에서 액세스할 수 있는 프로젝트 목록을 검색합니다. 프로젝트 폴더를 열어 해당 파일 및 하위 폴더를 봅니다.

   >[!NOTE]
   >
   >* 프로젝트 폴더는 최상위 수준에서 읽기 전용입니다. 프로젝트 폴더 자체는 이름을 바꾸거나, 삭제하거나, 이동할 수 없습니다.
   >* 프로젝트 폴더 내의 파일 및 폴더(열기, 편집, 생성, 삭제 등)로 작업할 수 있습니다.
   >* 레거시 Workfront 프로젝트는 Adobe Cloud Drive에 표시되지 않습니다. Adobe 클라우드 저장소에 저장된 프로젝트만 사용할 수 있습니다.

### 오류 툴팁

동기화 오류 또는 문제가 발생하면 파일 또는 폴더 아이콘 위로 마우스를 가져가 문제를 설명하는 도구 설명을 확인합니다.

| 오류 범주 | 툴팁 | 의미 |
|---|---|---|
| 동기화가 제외됨 | 지원되지 않는 파일 유형 | Adobe Cloud Drive에서 파일 형식을 지원하지 않습니다. |
| 동기화가 제외됨 | 지원되지 않는 파일 이름 | Adobe Cloud Drive에서 파일 이름을 지원하지 않습니다. |
| 동기화가 제외됨 | 상위 프로젝트 삭제됨 | 상위 Workfront 프로젝트가 삭제되었습니다. |
| 동기화 일시 중단됨 | 지원되지 않는 파일 콘텐츠 | 파일 콘텐츠를 동기화할 수 없습니다(예: 보안 문제가 감지됨). |
| 동기화 일시 중단됨 | 파일 이름에 잘못된 문자가 있습니다 | 파일 이름에 잘못된 문자가 있습니다. |
| 동기화 일시 중단됨 | 전체 경로가 너무 깁니다. | 파일 경로가 허용된 최대 길이를 초과합니다. |
| 동기화 일시 중단됨 | 쓰기 권한 없음 | 이 파일 또는 프로젝트에 대한 쓰기 권한이 취소되었습니다. |
| 동기화 오류 | 인증 문제 | 로그인 자격 증명에 문제가 있습니다. |
| 동기화 오류 | 클라우드 스토리지를 사용할 수 없음 | Adobe 클라우드 서비스를 일시적으로 사용할 수 없습니다. |
| 동기화 오류 | 클라우드 스토리지 가득 참 | 클라우드 스토리지 할당량이 꽉 찼습니다. |
| 동기화 오류 | 로컬 디스크가 꽉 참 | 로컬 디스크에 사용 가능한 공간이 부족합니다. |
| 동기화 오류 | 인터넷 연결 없음 | 장치가 인터넷에 연결되어 있지 않습니다. |
| 동기화 오류 | 예기치 않은 오류 | 동기화하는 동안 예기치 않은 오류가 발생했습니다. |
| 동기화 오류 | 계정 차단됨 | 서비스에 의해 계정이 차단되었습니다. |

>[!NOTE]
>
>연결 해제, 인증 실패, 네트워크를 사용할 수 없음, 로컬 디스크 가득 참 또는 클라우드 저장소 가득 참 등의 시스템 수준 오류는 개별 파일이 아닌 시스템 트레이(Windows) 또는 메뉴 모음(Mac)에 표시됩니다.

## 파일 열기

1. Adobe Cloud Drive에서 파일로 이동합니다.
1. 파일을 두 번 클릭합니다.

   기본 응용 프로그램에서 파일이 열립니다.

Adobe Cloud Drive는 다음을 포함하여 컴퓨터에 설치된 응용 프로그램이 열 수 있는 모든 파일 형식을 지원합니다.

* Adobe Creative Cloud 형식(PSD, AI, INDD, PRPROJ, AEP 등)
* Microsoft Office 문서(DOCX, XLSX, PPTX)
* 이미지(JPG, PNG, GIF 등)
* 비디오 파일(MP4, MOV 등)

>[!NOTE]
>
>클라우드 문서 형식(PSDC, AIDC 등)은 Adobe Cloud Drive를 통해 액세스할 때 표준 등가물(PSD, AI 등)로 열립니다.

## 파일 편집 및 저장

1. Adobe Cloud Drive에서 파일을 엽니다.
1. 응용 프로그램에서 변경 작업을 수행합니다.
1. **파일** > **저장**&#x200B;을 선택하거나 Ctrl+S(Windows) 또는 Cmd+S(Mac)를 눌러 파일을 저장합니다.

   변경 사항이 Adobe 클라우드 스토리지에 자동으로 동기화됩니다.

>[!IMPORTANT]
>
>**파일** > **저장** 또는 바로 가기 키를 사용하여 파일을 저장합니다. 드라이브에 중복 파일이 생성되므로 **다른 이름으로 저장**&#x200B;을 사용하여 복사본을 만들지 마십시오.

## 새 파일 만들기 또는 추가

프로젝트에서 직접 새 파일을 만들거나 로컬 저장소에서 기존 파일을 추가할 수 있습니다.

### 응용 프로그램에서 새 파일 만들기

1. 파일을 만드는 데 사용할 응용 프로그램을 엽니다.
1. 평소대로 파일을 만듭니다.
1. 저장할 때 Adobe Cloud Drive 프로젝트 폴더 내의 위치를 선택합니다.

   파일이 Adobe Cloud Drive에 나타나고 Adobe 클라우드 저장소에 동기화됩니다.

### 프로젝트에 기존 파일 추가

1. Finder(Mac) 또는 파일 탐색기(Windows)에서 Adobe Cloud Drive의 프로젝트 폴더를 엽니다.
1. 로컬 저장소에서 프로젝트 폴더로 파일을 끌거나 복사합니다.

   파일은 Adobe 클라우드 스토리지에 자동으로 동기화됩니다.

## 파일 및 폴더를 오프라인에서 사용 가능하도록 설정

파일 또는 폴더를 오프라인에서 사용할 수 있는 경우 인터넷 연결 없이 열고 편집할 수 있습니다. 오프라인 파일은 로컬 디스크 공간을 사용합니다.

### 장치에 파일 또는 폴더 보관

1. Adobe Cloud Drive의 파일 또는 폴더를 마우스 오른쪽 단추로 클릭합니다.
1. **이 장치에서 항상 유지**&#x200B;를 선택합니다.

   파일 또는 폴더가 로컬 캐시에 다운로드되므로 오프라인 상태에서도 로컬 캐시로 작업할 수 있습니다.

### 오프라인 액세스를 제거하여 공간을 확보하십시오.

1. 오프라인 파일 또는 폴더를 마우스 오른쪽 단추로 클릭합니다.
1. **사용 가능한 공간**&#x200B;을 선택하세요.

   파일 또는 폴더가 클라우드 저장소에 남아 있지만 로컬 캐시에서 제거됩니다.

>[!NOTE]
>
>오프라인 파일 및 폴더는 로컬 디스크 공간을 사용합니다. 더 이상 공간을 확보할 필요가 없는 파일 및 폴더에 대한 오프라인 액세스를 제거합니다.

## 로컬 저장소에 파일 복사

Adobe Cloud Drive의 파일을 로컬 드라이브에 복사할 수 있습니다. 원본은 Adobe Cloud Drive에 남아 있으며 복사본은 독립 로컬 파일이 됩니다.

1. Adobe Cloud Drive에서 파일을 마우스 오른쪽 단추로 클릭합니다.
1. **복사**&#x200B;를 선택한 다음 파일을 원하는 로컬 드라이브의 위치에 붙여넣습니다.

   파일이 대상에 복사됩니다. 원본은 Adobe Cloud Drive에 유지됩니다.

>[!NOTE]
>
>로컬 저장소에 복사된 파일은 독립 복사본입니다. 로컬 사본에 적용한 변경 사항은 Adobe 클라우드 스토리지에 다시 동기화되지 않습니다.

## Adobe 클라우드 드라이브에서 로그아웃

Adobe Cloud Drive에서 로그아웃해도 드라이브는 Finder 또는 파일 탐색기에 계속 표시됩니다. 그러나 로그아웃하는 동안 드라이브 내에서 변경한 사항과 로그아웃 전에 동기화되지 않은 변경 사항은 클라우드와 동기화하지 마십시오.

다음에 발생하는 일은 다시 로그인하는 계정에 따라 다릅니다.

### 동일한 계정으로 다시 로그인

Adobe Cloud Drive는 로그아웃할 때 로컬 탑재 폴더를 유지합니다. 동일한 자격 증명으로 다시 로그인하는 경우:

* Adobe Cloud Drive는 기존 마운트를 자동으로 재사용합니다.
* 로그아웃 전에 수행한 동기화되지 않은 변경 사항은 유지되며 연결이 복원되면 동기화됩니다.
* 사용자의 작업은 필요하지 않습니다.

### 다른 계정으로 로그인

로그아웃한 후 다른 Adobe 계정으로 로그인하는 경우:

* 현재 마운트된 폴더의 이름이 자동으로 변경되고 백업됩니다. 백업 폴더 이름은 `Adobe Cloud Drive <usermail>_<short_guid> (backup yyyy-MM-dd HH-mm-ss)` 형식을 사용합니다.
* 새 계정에 매핑된 Adobe 클라우드 드라이브는 Finder 또는 파일 탐색기에서 평소대로 사용할 수 있습니다.
* 백업 폴더를 제거하기 전에 동기화되지 않은 모든 작업을 수동으로 복구할 수 있습니다.

>[!NOTE]
>
>백업 폴더는 Mac의 `~/Library/CloudStorage` 및 Windows의 `C:\Users\<user>\`에 저장됩니다. 계정을 여러 번 전환하면 여러 개의 타임스탬프가 지정된 백업 폴더가 만들어집니다. 정기적으로 백업을 검토하고 정리하여 디스크 공간을 확보합니다.

## 파일 충돌 해결

충돌은 다음 상황 중 하나에서 발생할 수 있습니다.

* 여러 사용자가 동일한 파일을 동시에 편집하거나 삭제합니다.
* 다른 사용자가 파일을 여는 동안 파일이 수정됩니다.
* 네트워크 중단으로 인해 동기화 문제가 발생합니다.

### Adobe Cloud Drive로 충돌을 해결하는 방법

Adobe Cloud Drive는 충돌에 대해 복제 전략을 사용합니다.

* **파일 잠금이 없습니다.** 여러 사용자가 동시에 파일을 편집할 수 있습니다.
* **자동 복제** Adobe Cloud Drive에서 충돌을 감지하면 두 버전을 모두 유지합니다.
* **명명 지우기** 충돌 파일에는 사용자 이름과 타임스탬프가 `filename (Conflicted copy from username on date_time).extension` 형식으로 포함되어 있습니다. 예: `hero-banner (Conflicted copy from John on 2026-02-10_16-06-44).psd`.

### 수동으로 충돌 해결

1. 충돌 파일을 식별합니다. 충돌 파일의 파일 이름에 &quot;충돌하는 복사본&quot;이 있습니다.
1. 두 버전을 모두 검토하여 올바른 버전을 확인하십시오.
1. 올바른 버전을 유지하고 다른 버전을 삭제합니다.
1. 보존된 파일에 적절한 이름을 지정합니다.

>[!TIP]
>
>충돌을 최소화하려면
>
>* 파일을 편집하기 전에 동기화 상태를 확인하십시오.
>* 팀 구성원과 누가 어떤 파일을 편집하고 있는지에 대해 의견을 교환합니다.
>* 변경 사항이 신속하게 동기화되도록 자주 저장합니다.
>* 편집을 마치면 파일을 닫습니다.
