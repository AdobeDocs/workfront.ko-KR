## FAQ

**에 대해 자세히 알아보려면 어떻게 해야 합니까? [!DNL Adobe Admin Console]?**

* [[!DNL Adobe Admin Console] 개요](https://helpx.adobe.com/enterprise/using/admin-console.html)

**고객은 마이그레이션을 용이하게 하기 위해 어떻게 해야 합니까?**

마이그레이션을 예약하기 위해 기존 고객에게 연락됩니다. 마이그레이션 팀 지원 동료는 이 프로세스를 안내하고 [!DNL Admin Console] 설정을 지정하고, 가능한 한 쉽고 편리하게 이동하는 데 필요한 설명서 링크를 제공합니다.

* [[!DNL Adobe Workfront] 지원 개요](https://experienceleague.adobe.com/docs/customer-one/using/workfront/overview.html)
* [[!DNL Workfront Admin Console] 정보](https://experienceleague.adobe.com/docs/customer-one/using/workfront/landing.html)
* [[!DNL Adobe Business Platform] and [!DNL Admin Console] FAQ](https://experienceleague.adobe.com/docs/customer-one/using/workfront/faq.html)

**처리 방법 [!DNL Adobe Admin Console] 페더레이션 ID에 대해 이 기능이 이미 설정된 회사의 경우 [!DNL Workfront] SSO가 설정되어 있습니까?**

[!DNL Adobe Admin Console] 에는 다음과 같은 옵션이 있습니다 [!DNL Workfront]SSO를 IMS로 바꾸기 모든 사용자 프로비저닝은 [!DNL Admin Console], 및 사용자는 [!DNL Adobe] 로그인 화면 [!DNL Experience Cloud] 그들이 보게 될 곳 [!DNL Workfront] 옵션으로(액세스 권한이 부여된 경우)

**이 기능은 용으로 이미 AEM 관리 패널이 있는 고객에게 어떤 영향을 줍니까 [!DNL Adobe Assets] 하지만 SSO는[!DNL Workfront?]**

한 번 [!DNL Workfront] 가 [!DNL Admin Console] 응용 프로그램에 대해 다른 작업을 수행하지 않아도 됩니다 [!DNL Workfront] 기존 SSO 설정을 활용하려면 [!DNL Adobe Assets].

**SSO로 설정된 항목에 어떤 영향을 줍니까?**

SSO는 [!DNL Admin Console] 그리고 [!DNL Workfront] 응용 프로그램.

**내부 SSO입니까? [!DNL Active Directory] 여전히 IMS에 대한 옵션이 될 것입니까?**

IMS는 SSO의 대체이며 기능은 대부분 동일합니다. 모든 사용자 권한은에서 부여되고 제공됩니다 [!DNL Adobe Admin Console]로 설정되면 사용자는 [!DNL Adobe] &quot;&quot;을(를) 선택할 수 있는 로그인 화면[!UICONTROL 개인 계정]&quot; 또는 &quot;[!UICONTROL 회사 계정]로그인하려면 [!DNL Active Directory]로 설정되면 대부분 회사 계정으로 로그인하게 됩니다.

**SSO를 사용하지 않는 사용자의 경우 [!DNL Workfront] 로그인 URL을 변경하시겠습니까?**

로그인 URL이 변경됩니다. 그러나 이전 URL은 새 로그인 URL로 리디렉션되므로 이동할 사용자를 다시 교육해야 합니다.

**우리가 설정한 별칭이 여전히 작동합니까 아니면 [!DNL Workfront] 이 마이그레이션으로 변경되는 링크**

[!DNL Workfront] 리디렉션/별칭을 사용하여 홈 페이지로 가져올 수 있습니다.

**리디렉션이 비활성화될 시간이 있습니까? 아니면 항상 my.company.workfront.com을 입력할 수 있을까요?**

항상 사용자 지정 URL을 사용할 수 있습니다. 이러한 링크를 클릭하면 로 이동합니다 [!DNL Workfront] 다른 URL을 표시합니다. 하지만 그게 더 낫다 [!DNL experience] experience.adobe.com에 로그인하고 내에서 링크를 책갈피로 지정합니다. [!DNL Experience Cloud]. 리디렉션이 적을수록 지연 시간/로드 시간이 줄어듭니다.

**요청 큐에 대한 직접 링크가 끊어지나요?**

모든 직접 링크는 새 URL 패턴으로 리디렉션해야 합니다. 그러나 사람에게 링크를 배포했다면 직접 링크를 활용하고 예상 페이지로 지연되는 것을 방지하기 위해 업데이트를 보내야 합니다.

**으로 마이그레이션할까요 [!DNL Experience Cloud] 전역적으로 또는 특정 사용자에 대해 선택할 수 있습니까(일부 사용자가 다른 Adobe 제품을 사용하지는 않음)?**

전체 [!DNL Workfront] 고객 계정이 마이그레이션됩니다. 사용자별로 수행할 수 없습니다.

**모두 [!DNL Workfront] 사용자는 [!DNL Experience Cloud]? 아니면 관리자만?**

예, 모든 사용자는 [!DNL Experience Cloud]. IMS 로그인이 SSO를 대체합니다. 매우 유사한 경험으로, 다른 로그인 화면일 뿐입니다.

**사용자가 자신의 [!DNL Adobe] 계정 [!DNL Workfront] 둘 다 이미 있는 경우 계정을 사용하십시오.**

예. 이에 대한 프로세스가 있으며 조직에서 IMS로 이동할 때가 되면 더 많은 세부 사항이 제공됩니다.

**에 나타나는 결과 [!DNL Workfront] 없는 사용자 [!DNL Adobe] 계정?**

에 대한 액세스 권한이 없는 사용자 [!DNL Adobe Admin Console] 에 [!DNL Workfront] 를 만들어야 합니다.[!UICONTROL 개인 계정]&quot; 또는 [!DNL Adobe] 로그인할 수 있는 ID 계정입니다. 이 작업은 관리자에게 전자 메일을 보내 요청을 승인하거나 거부하도록 하며, 관리자가 사용자가 가지고 있는 액세스 유형을 구성할 수도 있습니다. 로그인하면 experience.adobe.com으로 이동하여 이메일 주소를 입력한 다음 선택합니다 [!UICONTROL 개인 계정]. 여기에서 액세스할 수 있습니다 [!DNL Workfront].

**만약 우리가 아무것도 없다면? [!DNL Adobe] 이외의 다른 제품[!DNL Workfront?]**

여전히 조직이 [!DNL Adobe Unified Experience]. 이(가) 제공됩니다 [!DNL Adobe] 위에 나열된 이점 과 함께 ID입니다.

**Adobe에는 외부 사용자가 포함되어 있습니다 [!DNL Workfront] 인스턴스. 우리는 그들이 [!DNL Adobe]. 콘솔 내에서 액세스 권한을 어떻게 제한합니까?**

[!DNL Admin Console] 관리자는 사용자가 액세스할 수 있는 작업과 액세스할 수 없는 항목을 제어할 수 있습니다. 외부 사용자가 액세스를 원할 때마다 [!DNL Adobe] 관리자에게 이메일을 보내는 ID입니다. 그런 다음 관리자는 제품에 대한 액세스를 허용하거나 거부하고 해당 조직이 소유한 제품에 대해 액세스할 수 있는/액세스할 수 없는 항목을 정의할 수 있습니다. 그런 다음 [!DNL Workfront] 시스템 관리자는 [!UICONTROL 사용자] 영역 [!DNL Workfront] 외부 사용자에 대해 더 세분화된 권한을 부여할 수 있습니다.

**그룹 관리자는 [!DNL Workfront]. 로 이동 [!DNL Experience Cloud], 그룹 관리자가 여전히 사람을 만들 수 있습니까?**

예, 사용자는 여전히 다음을 통해 생성할 수 있습니다. [!DNL Workfront]. 이러한 사용자는에 추가할 수 있습니다. [!DNL Experience Cloud] 자동. 또한 그룹 관리자를 [!DNL Admin Console] 할당하도록 허용 [!DNL Workfront] 사용자에게 표시됩니다.

**전환 마감일이 어떻게 되죠 [!DNL Experience Cloud]?**

현재 이동해야 할 마감일이 없습니다 [!DNL Adobe Experience Cloud]. Dell은 고객과 협력하여 고객이 이동할 준비가 되었을 때 선택할 수 있도록 노력하고 있습니다.

**지원 팀이 이 변경 사항을 위해 어떤 작업을 수행해야 합니까?**

지원 팀이 새 사용자를 만들 책임이 있는 경우 해당 사용자는 [!DNL Admin Console] 사용자를 만들고 Workfront에 대한 권한을 할당하는 데 사용되는 인터페이스입니다. 그렇지 않으면 내부 지원 팀에 큰 변화가 없을 것입니다.

**API 함수를 통해 보유한 통합에 어떤 영향을 줍니까?**

기존 URL 경로는 API 트래픽에 계속 사용할 수 있습니다. 통합에서 끝점을 업데이트하는 데 아무 작업도 수행하지 않아도 됩니다. 그러나 사용자 이름과 암호를 통해 직접 로그인하는 것은 지원되지 않습니다. 단, API 키를 사용해야 합니다(단, 단, [!DNL Workfront Fusion] 커넥터.

**그럼 [!DNL Creative Cloud] 사용자? 마이그레이션은 어떻게 영향을 줍니까? 그들에게는 어떤 이점이 있나요?**

에는 영향을 주지 않습니다 [!DNL Creative Cloud] 로 마이그레이션된 사용자 [!DNL Adobe Unified Experience].
