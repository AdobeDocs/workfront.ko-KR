# [!DNL Adobe Unified Experience] FAQ

다음 [!DNL Adobe Unified Experience] 대상 [!DNL Workfront] 을(를) 통해 [!DNL Adobe] 한 번의 로그인으로 한 곳에 애플리케이션을 배치할 수 있습니다. 다음 [!DNL Adobe] 탐색 영역은 와 원활하게 통합됩니다. [!DNL Workfront]. 몇 가지 기능이 서로 다르므로 다음과 같은 몇 가지 질문이 있을 수 있습니다. [!DNL Workfront] 인스턴스가 통합 경험으로 마이그레이션됩니다.

에 로그인하는 방법에 대한 자세한 내용은 [!DNL Adobe Unified Experience], 참조 [[!DNL Adobe Unified Experience] 대상 [!DNL Workfront]](/help/quicksilver/workfront-basics/navigate-workfront/workfront-navigation/adobe-unified-experience.md).

## 비교 [!DNL Adobe Unified Experience] 및 [!DNL Workfront only] 경험

를 사용하는 고객만 [!DNL Adobe Business Platform] / [!DNL Adobe Admin Console] 액세스 가능 [!DNL Adobe Unified Experience]. 아직 마이그레이션하지 않은 고객에게는 [!DNL Workfront only] 경험(전환 기능 없음) [!DNL Adobe] 응용 프로그램.

이 표에서는 두 경험 간에 차이점이 있는 몇 가지 기능에 대해 설명합니다.

| [!DNL Adobe Unified Experience] | [!DNL Workfront] 경험만 |
| ---- | ----|
| [!UICONTROL [!DNL Workfront] 메인 메뉴] 왼쪽에 있습니다. ![메인 메뉴](assets/main-menu-icon-left-nav.png) | [!UICONTROL [!DNL Workfront] 메인 메뉴] 오른쪽입니다 ![메인 메뉴](assets/main-menu-icon.png) |
| 모든 사용자가 단일 로그인 URL을 사용할 수 있습니다 [!DNL Adobe Experience Cloud] 응용 프로그램 | 에 로그인 [!DNL Workfront] 사용자 지정 포함 [!DNL Workfront] URL |
| 조직 전환기를 사용하면 다음 사이를 이동할 수 있습니다. [!DNL Workfront] 조직 및 환경 | 조직 전환기를 사용할 수 없습니다. |
| 탐색에는 의 최상위 탐색 영역이 포함되어 있습니다. [!DNL Adobe] products, [!DNL Adobe] 알림, 도움말 및 사용자 프로필 [!DNL Workfront] 탐색 막대 | 탐색에는 다음이 포함됩니다. [!DNL Workfront] 탐색 모음만 |

## FAQ(자주 묻는 질문)

### 통합 경험 Adobe 또는 Adobe Workfront 사용 여부를 어떻게 결정합니까?

조직이 Adobe 통합 환경에 있는지 확인하려면 Workfront에 액세스하는 데 사용하는 URL을 검사하십시오.

| URL | Adobe 경험 |
|------------|------------|
| (CompanyName).my.workfront.com | Workfront 경험 |
| experience.adobe.com | 통합 경험 Adobe |

### 에 대해 자세히 알아보려면 어떻게 해야 합니까? [!DNL Adobe Admin Console]?

에 대한 자세한 내용은 [!DNL Admin Console], 다음 문서를 검토하십시오.

* [다음을 준비합니다. [!DNL Adobe Admin Console]](/help/quicksilver/administration-and-setup/adobe-admin-console/prep-for-admin-console.md)
* [플랫폼 기반 관리의 차이점 ([!DNL Adobe Workfront]/[!DNL Adobe Business Platform])](/help/quicksilver/administration-and-setup/get-started-wf-administration/actions-in-admin-console.md)
* [[!DNL Adobe Admin Console] 개요](https://helpx.adobe.com/enterprise/using/admin-console.html)

### 마이그레이션을 용이하게 하기 위해 고객에게 필요한 것은 무엇입니까?

마이그레이션을 예약하려면 기존 고객에게 연락해야 합니다. 마이그레이션 팀 지원 동료가 고객에게 프로세스를 안내하고 다음을 조언합니다. [!DNL Admin Console] 를 설정하고, 가능한 한 단순하고 간편한 이동에 필요한 설명서 링크를 제공합니다.

* [[!DNL Adobe Workfront] 지원 개요](https://experienceleague.adobe.com/docs/customer-one/using/workfront/overview.html)
* [[!DNL Workfront Admin Console] 정보](https://experienceleague.adobe.com/docs/customer-one/using/workfront/landing.html)
* [[!DNL Adobe Business Platform] 및 [!DNL Admin Console] FAQ](https://experienceleague.adobe.com/docs/customer-one/using/workfront/faq.html)

### 어떻게 대처하고 있습니까 [!DNL Adobe Admin Console] federated ID에 대해 이 기능이 이미 과는 다르게 활성화되어 있는 회사의 경우 [!DNL Workfront] SSO가 설정되어 있습니까?

[!DNL Adobe Admin Console] 을 포함할 옵션이 있습니다. [!DNL Workfront], SSO를 IMS로 바꿉니다. 모든 사용자 프로비저닝은 [!DNL Admin Console], 사용자에게 다음이 표시됩니다. [!DNL Adobe] 로그인 화면: [!DNL Experience Cloud] 표시되는 위치 [!DNL Workfront] (액세스 권한이 부여된 경우).

### 이미 AEM 관리 패널을 가지고 있는 고객에게 어떤 영향을 줍니까? [!DNL Adobe Assets] - 그러나 SSO가 과(와) 다르게 구성되었습니다. [!DNL Workfront?]

한 번 [!DNL Workfront] 이(가) 다음으로 추가됨 [!DNL Admin Console] 애플리케이션, 다음 작업을 위해 다른 작업을 수행하지 않아도 됩니다. [!DNL Workfront] 에 대해 가지고 있는 기존 SSO 설정을 활용하려면 [!DNL Adobe Assets].

### 이 경우 SSO를 통해 설정된 항목에 어떤 영향을 줍니까?

SSO가 다음에 설정되어 있습니다. [!DNL Admin Console] 상속자: [!DNL Workfront] 응용 프로그램.

### SSO가 내부에 있음 [!DNL Active Directory] 아직도 IMS에서 선택사항이 될까요?

IMS는 SSO를 대체하는 것으로 기능은 대부분 동일합니다. 모든 사용자 권한은에서 부여되고 프로비저닝됩니다. [!DNL Adobe Admin Console], 그리고 사용자에게 다음이 표시됩니다 [!DNL Adobe] 선택할 수 있는 로그인 화면 &quot;[!UICONTROL 개인 계정]&quot; 또는 &quot;[!UICONTROL 회사 계정]&quot; 로그인합니다(있는 경우). [!DNL Active Directory], 대부분 회사 계정으로 로그인합니다.)

### SSO를 사용하지 않는 사용자의 경우 [!DNL Workfront] 로그인 URL을 변경하시겠습니까?

로그인 URL이 변경됩니다. 그러나 이전 URL은 새 로그인 URL로 리디렉션되므로 사용자를 어디로 이동할지 다시 교육해야 합니다.

### 설정된 별칭이 계속 작동합니까, 아니면 [!DNL Workfront] 링크가 이 마이그레이션과 함께 변경됩니까?

[!DNL Workfront] 리디렉션/별칭은 홈 페이지로 이동할 수 있습니다.

### 리디렉션이 비활성화되는 시간이 있습니까? 또는 항상 my.company.workfront.com에 입력할 수 있습니까?

언제든지 사용자 지정 URL을 사용할 수 있습니다. 그러한 링크를 클릭하면 다음 위치로 이동합니다. [!DNL Workfront] 다른 URL을 표시합니다. 하지만, 그것은 더 낫다 [!DNL experience] experience.adobe.com에 로그인하고 내에서 링크를 책갈피에 추가 [!DNL Experience Cloud]. 리디렉션을 줄이면 지연 시간/로드 시간이 줄어듭니다.

### 요청 대기열로 연결되는 직접 링크가 끊어집니까?

모든 직접 링크는 새 URL 패턴으로 리디렉션해야 합니다. 하지만 사람들에게 링크를 배포한 경우 직접 링크를 활용하고 예상 페이지로 이동하는 데 지연을 방지하기 위해 업데이트를 보내야 합니다.

### 다음으로 마이그레이션하시겠습니까? [!DNL Experience Cloud] 전역적으로 선택할 수 있습니까, 아니면 특정 사용자를 선택할 수 있습니까(모든 사용자가 다른 Adobe 제품을 사용하는 것은 아님)?

전체 [!DNL Workfront] 고객 계정이 마이그레이션됩니다. 이 작업은 사용자별로 수행할 수 없습니다.

### 모두 윌 [!DNL Workfront] 사용자는 를 통해 로그인해야 합니다. [!DNL Experience Cloud]? 아니면 관리자만?

예, 모든 사용자가 를 통해 로그인합니다. [!DNL Experience Cloud]. IMS 로그인이 SSO를 대체합니다. 로그인 화면만 다를 뿐 매우 유사한 경험입니다.

### 사용자가 다음을 연결해야 함 [!DNL Adobe] 계정 [!DNL Workfront] 둘 다 이미 가지고 있다면 계산하시겠습니까?

예. 이에 대한 프로세스가 있으며 조직이 IMS로 이동할 때 자세한 내용이 제공됩니다.

### 에 발생하는 결과 [!DNL Workfront] 권한이 없는 사용자 [!DNL Adobe] 계정?

에서 액세스 권한이 부여되지 않은 사용자 [!DNL Adobe Admin Console] 로 이동하려면 [!DNL Workfront] 은(는) &quot;을(를) 만들어야 합니다.[!UICONTROL 개인 계정]&quot; 또는 [!DNL Adobe] 로그인할 수 있는 ID 계정입니다. 이를 통해 관리자에게 이메일을 보내 요청을 승인하거나 거부하고, 관리자가 해당 사용자가 보유한 액세스 유형을 구성할 수 있습니다. 로그인하면 experience.adobe.com으로 이동하여 이메일 주소를 입력하고 을(를) 선택합니다. [!UICONTROL 개인 계정]. 여기에서 액세스할 수 있습니다. [!DNL Workfront].

### 아무것도 없으면 어떡하죠 [!DNL Adobe] 기타 제품 [!DNL Workfront?]

여전히 조직으로 마이그레이션하는 것이 좋습니다. [!DNL Adobe Unified Experience]. 다음을 받게 됩니다. [!DNL Adobe] 위에 나열된 혜택과 함께 ID를 제공합니다.

### 에 외부 사용자가 포함되어 있습니다. [!DNL Workfront] 인스턴스. 우리는 그들이 포함된 다른 제품에 액세스하지 않기를 바랍니다. [!DNL Adobe]. 콘솔 내에서 액세스를 어떻게 제한합니까?

[!DNL Admin Console] 는 관리자가 액세스할 수 있는 사용자와 액세스할 수 없는 사용자를 다양하게 제어할 수 있도록 해 줍니다. 외부 사용자가 액세스하려고 할 때마다 [!DNL Adobe] 관리자에게 이메일을 보내는 ID. 그런 다음 관리자는 제품에 대한 액세스를 수락 또는 거부하고 해당 조직이 소유한 제품에 대해 액세스할 수 있거나 액세스할 수 없는 항목을 정의할 수 있습니다. 그런 다음 [!DNL Workfront] 시스템 관리자는 [!UICONTROL 사용자] 영역 [!DNL Workfront] 외부 사용자에 대해 더 세분화된 권한을 만듭니다.

### 그룹 관리자는 다음 위치에서 사람을 만드는 데 사용됩니다. [!DNL Workfront]. 로 이동 [!DNL Experience Cloud], 그룹 관리자가 여전히 사람을 만들 수 있습니까?

예. 다음을 통해 사용자 생성이 가능합니다. [!DNL Workfront]. 이러한 사용자는에 추가할 수 있습니다. [!DNL Experience Cloud] 자동으로 표시됩니다. 에서 그룹 관리자를 제품 소유자로 설정할 수도 있습니다. [!DNL Admin Console] 할당하도록 허용 [!DNL Workfront] (으)로 가져왔습니다.

### 몇 시까지로 전환할 수 있나요 [!DNL Experience Cloud]?

현재 이동할 기한이 없습니다. [!DNL Adobe Experience Cloud]. 고객이 이동할 준비가 되면 선택할 수 있도록 고객과 협력하고 있습니다.

### 우리 지원팀이 이 변화를 위해 어떤 조치를 취해야 할까요?

지원 팀이 새 사용자를 만드는 책임이 있는 경우 해당 팀에 익숙해져야 합니다 [!DNL Admin Console] 사용자를 만들고 Workfront에 권한을 할당하는 데 사용되는 인터페이스입니다. 그렇지 않으면 내부 지원 팀에 큰 변화가 없을 수 있습니다.

### 이 경우 API 기능을 통해 제공되는 통합에 어떤 영향을 미칩니까?

기존 URL 경로는 API 트래픽에 계속 사용할 수 있습니다. 통합에서 끝점을 업데이트하기 위해 어떤 작업도 수행할 필요가 없습니다. 그러나 사용자 이름과 암호를 통한 직접 로그인은 지원되지 않습니다. API 키를 사용해야 합니다. 단, 다음과 같은 경우는 예외입니다. [!DNL Workfront Fusion] 커넥터.

### 다음에 대한 정보: [!DNL Creative Cloud] 사용자? 마이그레이션이 고객에게 어떤 영향을 미칩니까? 그들에게 이점이 있나요?

에 영향을 주지 않음 [!DNL Creative Cloud] (으)로 마이그레이션한 사용자 [!DNL Adobe Unified Experience].

### 다음에 대한 로그인이 변경됨: [!DNL Workfront] 모바일 사용자이십니까?

[!DNL Workfront] 모바일 사용자는 로의 마이그레이션의 영향을 받지 않아야 합니다. [!DNL Adobe Unified Experience].
