# [!UICONTROL 홈] 시작

<!--Audited: 12/2023-->

이 문서에서는 [!DNL Adobe Workfront]의 현재 [!UICONTROL Home] 영역에 대한 개요를 설명합니다. 새 [!UICONTROL 홈]에서 작업에 대한 자세한 내용은 [새 홈 시작](../new-home/get-started-with-new-home.md)을 참조하세요.

현재 [!UICONTROL Home] 영역에서 작업을 보는 방법에는 두 가지가 있습니다.

* [!UICONTROL 작업 목록]에서
* [!UICONTROL 일정]에서.

## [!UICONTROL 작업 목록]

[!UICONTROL 작업 목록]에서 할당된 모든 작업을 한 곳에서 볼 수 있습니다.

![[!UICONTROL 작업 목록] 패널](assets/worklist-and-right-panel-home.png)

위의 이미지에서 강조 표시된 대로 현재 [!UICONTROL Home] 영역에서 다음 섹션을 검토하십시오.

* **A**: 왼쪽 패널을 사용하여 자신에게 할당된 작업을 필터링하고 정렬하고 선택하십시오.

* **B**: 왼쪽 패널에서 작업 항목을 선택한 후 오른쪽 패널을 사용하여 사용자 지정 필드와 상호 작용하고 업데이트하며 시간을 기록합니다.

[!UICONTROL 홈 작업 목록] 사용에 대한 자세한 내용은 다음 문서를 참조하십시오.

* [[!UICONTROL Home] 영역의 [!UICONTROL 작업 목록]에 항목 표시](../../../workfront-basics/using-home/using-the-home-area/display-items-in-home-work-list.md)
* [[!UICONTROL Home] 영역에서 작업 항목 만들기](../../../workfront-basics/using-home/using-the-home-area/create-work-items-in-home.md)

## [!UICONTROL 일정]

작업 항목의 기한이 다가오면 표시하여 작업을 시각화할 수 있으며 [!UICONTROL Home] 영역의 [!UICONTROL Calendar]에서 작업을 검토하여 완료 시간을 차단할 수 있습니다.

![](assets/calendar-home-nwe-350x114.png)

위의 이미지에서 강조 표시된 대로 현재 [!UICONTROL Home] 영역에서 다음 섹션을 검토하십시오.

* **A**: [!UICONTROL 달력] 보기를 사용하여 완료해야 하는 작업의 시각적 맵을 만드십시오. 작업 시간을 차단하려면 [!UICONTROL 작업 목록]의 작업 항목을 클릭하여 [!UICONTROL 일정](으)로 드래그하기만 하면 됩니다.

* **B**: [!UICONTROL 일정]의 동기화 옵션을 사용하여 [!DNL Outlook] 일정과 동기화하세요. 회의를 관리하고 한 곳에서 모두 작업하십시오.

* **C**: [!UICONTROL 세부 정보] 단추를 사용하여 작업 항목에 대한 자세한 정보를 볼 수 있는 새 오른쪽 패널을 엽니다.

* **D**: 기한 표시줄을 사용하여 할당된 작업 기한이 언제인지 추적합니다.

홈 캘린더 사용 방법에 대한 자세한 내용은 [[!UICONTROL 홈 캘린더 사용] 보기](../../../workfront-basics/using-home/using-the-home-area/use-home-calendar-view.md)을 참조하십시오.

## 작업 찾기

[!UICONTROL 홈] 영역은 할당된 작업, 문제 및 승인을 찾는 원스톱 샵입니다. 필터, 그룹화 및 정렬 옵션은 함께 작동하여 원하는 방식으로 작업 항목을 구성할 수 있습니다. 그룹화를 사용하여 항목을 함께 그룹화하고, 필터를 사용하여 완료해야 하는 작업에 초점을 맞추고, 마지막으로 정렬을 사용하여 항목을 그룹화의 오름차순 또는 내림차순으로 나열할 수 있습니다.

>[!NOTE]
>
>필터 및 정렬 옵션은 브라우저에 저장됩니다. 동일한 컴퓨터에서 동일한 브라우저를 일관되게 사용하는 경우(그리고 사이트 데이터를 지우지 않는 경우) 필터 및 정렬은 변경되지 않지만 브라우저나 컴퓨터를 전환하는 경우 필터 및 정렬은 달라집니다.

### 그룹화를 사용하여 유사한 작업 항목 그룹화

그룹화를 사용하여 작업 목록의 공통 헤더 아래에 유사 항목을 표시할 수 있습니다. 다음과 같은 방법으로 항목을 그룹화할 수 있습니다.

* [!UICONTROL 계획된 완료]
* [!UICONTROL 계획된 시작]
* [!UICONTROL 커밋 일자]
* [!UICONTROL 프로젝트]
* [!UICONTROL 내 우선 순위]

그룹화를 사용하는 방법에 대한 자세한 내용은 [[!UICONTROL Home] 영역의 [!UICONTROL 작업 목록]에 항목 표시](../../../workfront-basics/using-home/using-the-home-area/display-items-in-home-work-list.md)를 참조하십시오.

![](assets/group-by-drop-down-expanded-in-home-with-planned-start-date-nwe-350x273.png)

### 필터로 초점 좁히기

[!UICONTROL 작업 목록] 필터를 사용하면 특정 작업 항목으로 포커스를 좁힐 수 있습니다.

다음은 홈에서 보는 항목을 필터링하는 방법의 예입니다.

* 항목 유형 및 상태별로 필터링합니다.

  예를 들어 모든 작업을 표시하려면 [!UICONTROL 작업] 필터를 선택할 수 있습니다. 이보다 더 자세히 알아보고 작업을 시작할 준비가 된 작업만 표시하려면 [!UICONTROL 작업] 필터에서 [!UICONTROL 시작할 준비] 필터를 선택하십시오.

* 항목 유형으로만 필터링합니다.

  예를 들어 [!UICONTROL 문제]를 선택하여 모든 상태의 모든 문제를 보거나([!UICONTROL 작업 중] 또는 [!UICONTROL 요청됨]) [!UICONTROL 승인]을 선택하여 모든 작업 항목, 액세스 요청, 타임시트, 문서 및 증명 승인을 볼 수 있습니다.

필터를 사용하는 방법에 대한 자세한 내용은 [[!UICONTROL Home] 영역의 [!UICONTROL 작업 목록]에 항목 표시](../../../workfront-basics/using-home/using-the-home-area/display-items-in-home-work-list.md)를 참조하십시오.

![](assets/displaying-work-items-filters-nwe-350x401.png)

### 내 팀에 할당된 액세스 작업

[!UICONTROL Home] 영역에는 팀에 할당된 작업 및 문제를 표시하는 영구 그룹화([!UICONTROL 팀 요청])가 있습니다. 이 섹션에는 팀이 표시됩니다. 팀 이름을 클릭하면 해당 팀의 팀 요청 페이지가 열립니다.

>[!NOTE]
>
>필터 및 정렬 옵션은 [!UICONTROL 팀 요청] 그룹화에 영향을 주지 않습니다. 이 그룹화는 팀에 할당된 작업이 있는 한 볼 수 있습니다.

팀 요청 액세스에 대한 자세한 내용은 [!UICONTROL Home] 영역](../../../workfront-basics/using-home/using-the-home-area/manage-work-and-team-requests-home.md)에서 [[!UICONTROL 관리] 작업 및 팀 요청 문서를 참조하십시오.

![](assets/team-requests-expanded-home-group-by-drop-down-nwe-350x314.png)

### 제출한 작업 모니터링

[!UICONTROL 작업 목록]에서 직접 승인을 위해 제출한 작업을 추적하세요. [!UICONTROL 작업 목록]에서 [!UICONTROL 승인]을(를) 보도록 선택하면 [!UICONTROL 내가 제출한 승인]에 대한 영구 그룹화가 있습니다. 승인자에게 승인이 필요한 작업을 알려 줄 수 있습니다. 필요한 경우 승인을 회수할 수도 있습니다.

>[!NOTE]
>
>필터 및 정렬은 [!UICONTROL 내가 제출한 승인] 그룹화에 영향을 주지 않습니다. 승인 대기 중인 작업이 있는 한 이 그룹화가 표시됩니다.
>
>[!UICONTROL 승인] 필터의 승인 수에 승인을 위해 제출한 항목의 수가 표시되지 않습니다. [!UICONTROL 승인] 필터에는 승인 대기 중인 항목만 표시됩니다.

![](assets/approvals-expanded-home-group-by-drop-down-nwe-350x415.png)

## 본인에게 중요한 것에 우선 순위 지정

[!UICONTROL 작업 목록]을(를) 사용하면 [!UICONTROL 내 우선 순위] 그룹화를 통해 자신에게 중요한 작업을 표시하고 우선 순위를 지정할 수 있습니다. [!UICONTROL 내 우선 순위] 목록에 최대 20개의 항목을 추가할 수 있습니다.

>[!NOTE]
>
>[!UICONTROL 내 우선 순위]에 추가된 모든 작업 항목은 귀하에게만 표시되며, 아무도 귀하의 작업 우선 순위를 지정할 수 없습니다.

[!UICONTROL 내 우선 순위]를 사용하는 방법에 대한 자세한 내용은 [[!UICONTROL 홈] 영역에서 작업 우선 순위 지정](../../../workfront-basics/using-home/using-the-home-area/prioritize-work-in-home.md)을 참조하세요.

![](assets/prioritizing-in-home-group-by-drop-down-nwe-350x412.png)

## 완료

[!UICONTROL 작업 목록]에서 항목 이름을 클릭한 후 [!UICONTROL Home]의 오른쪽 패널을 사용하여 작업을 업데이트하고 완료하십시오.

### 사용자 지정된 공간에서 작업

[!DNL Workfront] 관리자는 필요한 정보를 더 쉽게 찾고 업데이트할 수 있도록 [!UICONTROL 작업 목록]의 항목에 최대 16개의 사용자 지정 필드를 추가할 수 있습니다. 대부분의 필드는 편집할 수 있으므로 작업 항목으로 이동할 필요가 없습니다.

이 영역의 필드를 업데이트하려면 필드를 클릭하면 됩니다. 필드를 편집할 수 있는 경우 오른쪽 패널에서 수정할 수 있습니다.

>[!NOTE]
>
>이 필드가 레이아웃 템플릿의 작업 항목에 추가된 경우 [!UICONTROL 커밋 일자]를 업데이트할 수 없습니다.

![](assets/16-fields-home-350x137.png)

### 다른 사용자에게 중요한 정보 업데이트

[!UICONTROL Home] 영역에서 다른 사람과 빠르고 효율적으로 대화할 수 있습니다. 작업 항목의 댓글 달기 영역을 사용하여 질문을 하거나 다른 사용자에게 최근 업데이트를 알리거나 [!UICONTROL Home] 영역에서 직접 댓글에 답글을 달 수 있습니다. 또한 다음과 같은 방법으로 진행 상황에 대한 다른 정보를 업데이트할 수 있습니다.

* 댓글 영역에서 [!UICONTROL 커밋 일자]를 업데이트하는 중

  >[!NOTE]
  >
  >   홈에서 커밋 일자를 갱신할 수 있는 유일한 위치입니다.

* 댓글을 추가하거나 항목의 [!UICONTROL 조건]을 업데이트하여 진행 상황을 보고하는 중
* 완료율 막대 업데이트

[!UICONTROL Home] 영역에서 통신하는 방법에 대한 자세한 내용은 [작업 업데이트](../../../workfront-basics/updating-work-items-and-viewing-updates/update-work.md)를 참조하십시오.

![](assets/updates-home-350x198.png)

### 시간 기록

개별 작업 항목에서 작업한 시간을 추적합니다. [!UICONTROL 작업 시간]과 같은 일반 시간 유형을 선택하거나 [!DNL Workfront] 관리자가 설정한 프로젝트별 시간 유형을 선택할 수 있습니다.

[!UICONTROL Home] 영역에서 시간을 기록하는 방법에 대한 자세한 내용은 문서 [시간 기록](../../../timesheets/create-and-manage-timesheets/log-time.md)의 [Home](../../../timesheets/create-and-manage-timesheets/log-time.md#home) 섹션을 참조하십시오.

![](assets/log-time-home-350x181.png)
