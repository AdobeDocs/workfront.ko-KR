## 간판 보드의 WIP 제한 개선 사항 {#work-in-progress-wip-limit-enhancements-on-the-kanban-board}

### 간판 보드의 각 열에 대한 WIP 한도 구성

이제 간판 보드의 각 열에 대해 WIP(Work in Progress) 제한을 구성할 수 있습니다. 

이 변경 전에 간판 보드의 모든 열에 적용되는 하나의 WIP 제한만 구성할 수 있습니다. 

자세한 내용은 섹션을 참조하십시오  [진행 중인 작업(WIP) 제한 구성](../../../../agile/get-started-with-agile-in-workfront/configure-kanban.md#configur4) 기사  [간판 구성](../../../../agile/get-started-with-agile-in-workfront/configure-kanban.md).

### 간판 보드에서 WIP 한도 직접 갱신

이제 팀에 대한 편집 권한이 있는 팀 구성원은 간판 보드에서 직접 WIP 제한을 갱신할 수 있습니다.

이 변경 전에 팀 설정 영역에서만 WIP 제한을 업데이트할 수 있습니다.

자세한 내용은 문서 를 참조하십시오.

## 홈 영역에서 업데이트된 작업 목록(왼쪽 패널) {#updated-work-list-left-panel-in-the-home-area}

홈 영역의 작업 목록에는 다음 개선 사항이 포함되어 있습니다.

* 읽지 않은 항목이 이제 볼딩과 파란색 점이 돋보입니다.

   홈 영역 외부에서 보는 항목은 여전히 홈 영역에서 읽지 않음으로 표시됩니다.

   자세한 내용은 [홈 영역의 작업 목록에 항목을 표시합니다.](../../../../workfront-basics/using-home/using-the-home-area/display-items-in-home-work-list.md).

* 이제 문제 옆에 표시되는 문제 아이콘과 문제가 구별됩니다.
* 이제 승인 유형이 승인 유형으로 구분되며 승인 유형이 표시됩니다. 가능한 승인 유형은 작업, 프로젝트, 문제, 액세스, 문서, 작업표 및 증명입니다.

   이 변경 전에 승인은 &quot;Approval&quot;이라는 단어로만 구분되었습니다.

* 이제 항목이 시작할 준비가 되었는지 여부에 따라 구별됩니다. 가능한 옵션은 시작 준비, 준비 안 됨 또는 작동 중입니다.

   이 정보는 항상 승인을 시작할 준비가 되므로 승인을 위해 표시되지 않습니다.

* 문서가 항목에 첨부될 때마다 문서 아이콘이 항목의 이름 아래에 표시됩니다.
* 이제 작업 목록 내의 그룹화를 축소하고 확장하여 표시되는 정보를 더 잘 제어할 수 있습니다. 그룹화는 지연, 프로젝트, 날짜 및 완료입니다.

   기본적으로 이번 주 섹션이 확장되고 다른 모든 그룹이 축소됩니다.

* 계획 완료 일자별로 품목을 정렬할지 또는 커밋 일자별로 품목을 정렬할지 여부를 선택합니다.
* 그룹화된 각 항목의 수가 그룹 제목 옆에 괄호로 표시됩니다.

   이 번호는 완료 그룹화에 사용할 수 없습니다.

   자세한 내용은 [홈 영역의 작업 목록에 항목을 표시합니다.](../../../../workfront-basics/using-home/using-the-home-area/display-items-in-home-work-list.md).

* 드래그하여 놓기를 통해 작업 목록 크기를 조정합니다. 작업 목록의 크기를 조정하여 화면의 최대 절반을 소비할 수 있습니다. 설정한 크기는 다음에 홈에 액세스할 때 유지됩니다.

   이 변경 전에 작업 목록의 크기를 변경할 수 없습니다.

* 이제 요청에 대해 요청을 수행한 사용자의 사용자 아바타가 표시되고 텍스트 &quot;[Approver_name] 승인도 부탁드립니다.&quot;
* 새 개인 작업을 만들 때 이제 &quot;할 일&quot; 단추에 &quot;개인&quot;이라는 레이블이 지정됩니다.

   자세한 내용은 [홈 영역에서 작업 항목 만들기](../../../../workfront-basics/using-home/using-the-home-area/create-work-items-in-home.md) 기사 [홈 영역에서 작업 항목 만들기](../../../../workfront-basics/using-home/using-the-home-area/create-work-items-in-home.md).

* 지연 항목은 계획 완료 일자 이후 한 시간 후에만 지연 항목으로 표시됩니다.

홈 영역에 대한 자세한 내용은 [홈 영역 사용](../../../../workfront-basics/using-home/using-the-home-area/use-the-home-area.md).

## 대화형(리치 미디어) 콘텐츠를 교정할 수 있는 새 데스크탑 언어 교정 뷰어 {#new-desktop-proofing-viewer-for-proofing-interactive-rich-media-content}

새로운 데스크탑 언어 교정 뷰어를 사용하면 대화형 내용을 입증할 수 있습니다. 기존 레거시 교정 뷰어와 브라우저에서 실행되는 새 교정 뷰어와 달리, 데스크탑 교정 뷰어는 워크스테이션에서 실행되는 응용 프로그램입니다.

새로운 데스크탑 언어 교정 뷰어를 사용하기 전에 레거시 언어 교정 뷰어에서만 대화형 내용을 증명 할 수 있습니다. 

Desktop Proofing Viewer에는 대화형 내용을 교정할 수 있는 레거시 Proofing Viewer에 대한 다음과 같은 개선 사항이 포함되어 있습니다.

* 비보안(HTTP) 사이트 검토

   레거시 언어 교정 뷰어를 사용하여 보안(HTTPS) 사이트만 검토할 수 있습니다

* iframe으로 보호된 사이트(iframe 내에서 볼 수 없도록 보호된 사이트)를 검토합니다.

   레거시 언어 교정 뷰어는 iframe 내에서 볼 수 없도록 보호된 사이트에 대한 검토를 지원하지 않았습니다.

* 다양한 장치에 대해 사전 구성된 해상도로 콘텐츠를 봅니다. 예를 들어, 다양한 표준 데스크탑 해상도나 iPhone 8과 같은 개별 장치에서 컨텐츠가 표시되는 방식을 확인할 수 있습니다. 

데스크탑 언어 교정 뷰어 다운로드, 설치 및 사용에 대한 자세한 내용은 를 참조하십시오.

데스크탑 언어 교정 뷰어와 브라우저 기반 언어 교정 뷰어의 기능 차이에 대한 자세한 내용은 다음을 참조하십시오 [Web Proofing Viewer와 Desktop Proofing Viewer 개요의 차이점](../../../../review-and-approve-work/proofing/proofing-overview/understand-differences-between-web-viewer.md).

## 리소스 계획자에서 사용자 뷰 내보내기 {#export-the-user-view-in-the-resource-planner}

일부 성능 문제를 해결하기 위해 사용자 보기에 표시할 때 리소스 플래너에서 데이터 내보내기를 일시적으로 비활성화했습니다. 이 릴리스에서는 사용자 뷰에 리소스 계획자를 표시할 때 데이터 내보내기를 다시 활성화합니다.

## 시스템 설정: 외부 페이지의 세션 정보 {#system-setting-session-information-in-external-pages}

이제 Workfront 관리자는 외부 페이지를 만들 때 세션 정보(예: 세션 ID)의 사용을 제한할 수 있습니다.

이 변경 전에 외부 페이지를 만들 수 있는 사용자는 다른 사이트를 Workfront 대시보드에 포함할 때 세션 정보를 사용할 수 있습니다. 

Workfront에서 시스템 환경 설정 구성에 대한 자세한 내용은 [시스템 보안 환경 설정 구성](../../../../administration-and-setup/manage-workfront/security/configure-security-preferences.md).

이제 Workfront에서 Google 팀 드라이브에 있는 문서 또는 폴더를 연결할 수 있습니다.

## 이제 Internet Explorer 또는 Safari를 사용할 때 클립보드에서 붙여넣기 옵션이 흐리게 표시됩니다. {#paste-from-clipboard-option-now-displays-as-dimmed-when-using-internet-explorer-or-safari}

이제 Internet Explorer 또는 Safari 브라우저를 사용할 때 클립보드에서 붙여넣기 옵션이 흐리게 표시되고 Chrome 및 Firefox 브라우저만 이 기능에 대해 지원된다는 도구 설명이 제공됩니다.

이 변경 전에 Internet Explorer 또는 Safari를 사용할 때 이 옵션이 표시되지 않았습니다. 

클립보드에서 이미지 붙여넣기에 대한 자세한 내용은 다음을 참조하십시오 [클립보드에서 이미지 붙여넣기](../../../../documents/managing-documents/paste-image-clipboard.md).

## 새로운 기능과 함께 Android용 새로운 베타 환경 {#new-beta-environment-for-android-along-with-new-features}

이제 Adobe 팀이 모바일 앱을 위해 작업 중인 최신 기능을 Beta 테스터로 등록하여 일반 대중에게 릴리스하기 전에 경험할 수 있습니다. 이 환경은 현재 Android 휴대폰용 Workfront 모바일 앱에서만 지원됩니다.

Workfront 모바일 앱용 베타 테스터로 등록하는 방법에 대한 자세한 내용은 를 참조하십시오.

이제 모바일 앱의 베타 버전에서 다음 개선 사항을 사용할 수 있습니다.

* 새 계정 페이지

   이제 계정 정보를 보고, 모바일 설정을 관리하거나, 피드백을 제출하거나, 새 계정 페이지에서 로그아웃할 수 있습니다.

   이 개선 사항 전에 모바일 앱에서 계정 정보를 볼 수 없으며 설정에 액세스하고 피드백을 제출하고 Workfront 기본 메뉴에서 로그아웃할 수 있습니다.

* 새 탐색 아래쪽 패널

   이제 모바일 앱의 모든 영역을 시작하는 화면 맨 아래에서 보다 눈에 띄는 탐색 모음을 사용할 수 있습니다.

   이 개선 사항 전에 Workfront 기본 메뉴에 기능 영역이 나열됩니다. Workfront 기본 메뉴가 제거되어 새 아래쪽 탐색 막대로 대체되었습니다.

   새 탐색 모음 구성에 대한 자세한 내용은 의 &quot;모바일 앱 구성&quot; 섹션을 참조하십시오.

* 새 알림 목록 보기

   향상된 알림 목록 모양과 느낌을 사용하면 목록에 있는 알림의 종류 및 읽기 여부에 따라 쉽게 구분할 수 있습니다.

* 검색 상자가 더 눈에 띄는 위치로 이동되었습니다.

* 새 로그인 경험보다 간소화된 새로운 로그인 경험이 있습니다.

* 새로운 자습서 경험

   이제 사용자가 처음 로그인할 때 앱을 안내하는 새로운 자습서가 제공됩니다. 이 경험 이전에 자습서는 사용자가 특정 기능 영역에 액세스할 때만 실행됩니다.

## 이벤트 구독 메시지에 대한 필터의 예 {#examples-of-filters-for-event-subscriptions-messages}
