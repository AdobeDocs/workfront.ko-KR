## 홈 영역에서 제출한 승인 추적 {#track-approvals-you-ve-submitted-in-the-home-area}

이제 홈 영역을 사용하여 다른 사용자에게 제출한 승인을 상호 작용할 수 있습니다.

홈에서 다음을 수행할 수 있습니다.

* 승인 회수
* 사용자에게 승인 알림
* 문서 승인에 사용자 추가
* 증명 승인 보기

   이전에는 내 작업 영역에서만 제출된 승인 과 상호 작용할 수 있었습니다. 

자세한 내용은 [승인 관리](../../../../review-and-approve-work/manage-approvals/manage-approvals.md). 

## Workfront에서 Flash 기반 도구 교체 {#replacement-of-flash-based-tools-in-workfront}

Flash 기술을 사용하는 모든 기능은 2018.3 릴리스로 Workfront에서 제거될 예정이며 새 솔루션으로 대체됩니다.

이러한 기능 제거의 영향을 받는 모든 영역에 대한 자세한 내용을 알고 이를 대체할 도구를 보려면 를 참조하십시오 [Adobe Workfront에서 Flash 기반 도구 교체](../../../../product-announcements/announcements/announcement-archive/replace-flash-tools.md).

## 작업 라이센스 사용자가 프로젝트의 스케줄링 하위 탭에 액세스할 수 있도록 허용 {#allowing-work-license-users-to-access-the-scheduling-subtab-of-a-project}

작업 라이센스가 있는 사용자는 이제 스태핑 탭 아래의 프로젝트의 스케줄링 하위 탭에 액세스할 수 있습니다.

이 개선 사항 이전에는 계획 라이센스가 있는 사용자만 이 하위 탭에 액세스할 수 있었습니다.

프로젝트의 스케줄링 하위 탭에 액세스하는 데 필요한 액세스에 대한 자세한 내용은 &quot;자원 스케줄링 시작&quot;을 참조하십시오.

>[!NOTE]
>
>리소스 예약 도구는 23.1 릴리스의 Workfront에서 더 이상 사용되지 않으며 제거되었습니다. 작업 로드 밸런서를 사용하는 리소스 예약에 대한 자세한 내용은 [작업 로드 밸런서 개요](../../../../resource-mgmt/workload-balancer/overview-workload-balancer.md).

## 리소스 플래너의 필터 개선 {#improved-filters-in-the-resource-planner}

이제 리소스 플래너의 정보를 필터링하여 사용자 정의 필드를 비롯한 프로젝트별 및 사용자별 필드를 포함할 수 있습니다.

이 개선 사항을 수행하기 전에 Resource Planner에서 제한된 수의 기본 제공 필드 및 기준에 대해서만 필터링할 수 있습니다.

리소스 계획자의 정보 필터링에 대한 자세한 내용은 [리소스 계획자에서 정보 필터링](../../../../resource-mgmt/resource-planning/filter-resource-planner.md).

## 리소스 계획자에서 실제 시간 보기 {#view-actual-hours-in-the-resource-planner}

리소스 계획자의 사용자 뷰에서 실제 시간을 볼 수 있습니다.

이 개선 전에 실제 시간이 리소스 계획자에 표시되지 않았습니다.

리소스 계획자에서 시간 정보 보기에 대한 자세한 내용은 [리소스 플래너 개요](../../../../resource-mgmt/resource-planning/get-started-resource-planner.md).

## 비즈니스 사례 내보내기의 자원 계획자 데이터 {#resource-planner-data-in-the-business-case-export}

업무 사례를 PDF으로 내보낼 때 내보낸 파일의 재무 정보가 자원 예산 영역 및 자원 계획자의 정보를 반영합니다. 

이전에는 내보낸 파일이 이전 자원 예상 영역과 능력 계획자의 정보를 반영했습니다. 

비즈니스 사례 내보내기에 대한 자세한 내용은 [프로젝트에 대한 비즈니스 사례 생성](../../../../manage-work/projects/define-a-business-case/create-business-case.md) 섹션 [프로젝트에 대한 비즈니스 사례 생성](../../../../manage-work/projects/define-a-business-case/create-business-case.md).

## 회사 레벨 청구율 대체 사용 {#enable-company-level-billing-rate-override}

이제 회사 수준의 청구 요금이 프로젝트 수준의 청구 비율을 대체하도록 허용하는 옵션을 활성화할 수 있습니다. 회사 레벨에서 청구 비율을 갱신한 후 해당 변경 사항을 반영하도록 프로젝트에서 자금을 수동으로 재계산할 수 있습니다.

이 설정을 사용하면 회사 레벨 청구 단가가 프로젝트의 과거 작업 역할 비율을 대체합니다.

이전에는 프로젝트에서 회사를 제거한 다음 다시 첨부하지 않으면 회사 수준의 청구 비율 변경 사항을 프로젝트에 적용할 수 없었습니다.

자세한 내용은 [회사 수준 청구 단가로 프로젝트 레벨 청구 비율 대체](../../../../manage-work/projects/project-finances/override-project-level-with-company-level-billing-rates.md). 

## 간소화된 역할 기반 승인 {#simplified-role-based-approvals}

이제 역할 기반 승인이 프로젝트에서 보다 간편하게 작동합니다. 이제 사용자는 프로젝트 팀에 할당되고 사용자 프로필에 할당된 역할이 있어야 기본 또는 보조 역할에 할당된 승인을 볼 수 있습니다.

이전에는 사용자를 프로젝트 팀에 지정하고, 사용자 프로필에 올바른 역할이 있는지 확인하고, 이전 팀 빌더에서 해당 역할을 선택해야 했습니다.

자세한 내용은 섹션을 참조하십시오 [프로젝트 팀 개요](../../../../manage-work/projects/planning-a-project/project-team-overview.md) 기사 [프로젝트 팀 개요](../../../../manage-work/projects/planning-a-project/project-team-overview.md).

## 활용률 보고서: 신규 자원 예산 책정 영역에서 예산 책정된 시간 채우기 {#utilization-report-populates-budgeted-hours-from-new-resource-budgeting-area}

>[!NOTE]
>
> 이 기능은 2018.2 릴리스를 통해 미리 보기 환경에 처음 도입되었습니다. 2018.3 릴리스와 함께 프로덕션 환경에 릴리스됩니다. 

활용률 보고서의 예산책정된 시간은 이제 업무 사례의 새 자원 예산책정 영역에서 사용할 수 있는 정보로 채워집니다.

이 변경 전에 레거시 리소스 예상 영역의 정보가 사용되었습니다.

자세한 내용은  [자원 가동률 보고서 개요](../../../../reports-and-dashboards/reports/using-built-in-reports/resource-utilization-report.md) 기사  [자원 가동률 보고서 개요](../../../../reports-and-dashboards/reports/using-built-in-reports/resource-utilization-report.md).

## 활용률 보고서: 프로젝트에 대한 사용자별 예산책정 시간 보기 {#utilization-report-view-budgeted-hours-by-user-on-a-project}

>[!NOTE]
이 기능은 2018.2 릴리스를 통해 미리 보기 환경에 처음 도입되었습니다. 2018.3 릴리스와 함께 프로덕션 환경에 릴리스됩니다. 

이제 프로젝트에 대한 가동률 보고서에 사용자별 예산책정된 시간이 표시됩니다.

이 변경 전에 가동률 보고서는 Job 역할별로 예산책정된 시간만을 표시합니다. 

자세한 내용은  [자원 가동률 보고서 개요](../../../../reports-and-dashboards/reports/using-built-in-reports/resource-utilization-report.md) 기사  [자원 가동률 보고서 개요](../../../../reports-and-dashboards/reports/using-built-in-reports/resource-utilization-report.md).

## 언어 교정 뷰어에서 더 이상 SWF 파일이 지원되지 않음 {#swf-files-no-longer-supported-in-the-proofing-viewer}

Adobe Flash과 관련된 보안 문제로 인해 교정 뷰어에서 더 이상 SWF 파일에서 만든 증명을 지원하지 않습니다. SWF 파일에 대해 이전에 만든 증명을 열면 교정 뷰어에 이전에 증명 콘텐츠가 표시된 위치에 메시지가 표시됩니다. 증명서에 나오는 모든 주석을 볼 수 있습니다. 주석을 다운로드하고 원본 SWF 파일을 다운로드할 수도 있습니다.

자세한 내용은 [지원되는 언어 교정 파일 형식 및 크기 제한 개요](../../../../review-and-approve-work/proofing/proofing-overview/supported-proofing-file-types.md) 기사 [지원되는 언어 교정 파일 형식 및 크기 제한 개요](../../../../review-and-approve-work/proofing/proofing-overview/supported-proofing-file-types.md). 

## 언어 교정 뷰어의 주석 목록에서 작업을 보다 빠르게 액세스할 수 있습니다. {#actions-more-quickly-accessible-in-the-comment-list-in-the-proofing-viewer}

이제 교정 뷰어에서 주석 작업을 보다 빠르게 추가하거나 변경할 수 있습니다. 주석에서 자세히 아이콘을 클릭한 다음 드롭다운 메뉴에서 원하는 작업을 클릭하면 됩니다.

이전에는 설명을 열고 드롭다운 메뉴를 열고 하위 메뉴를 연 다음 원하는 작업을 클릭해야 했습니다.

자세한 내용은 [증명 주석에 작업 사용](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/comment-on-a-proof/use-actions-on-comments-in-viewer.md).

## 교안 뷰어의 증명 공유 개선 {#proof-sharing-improvement-in-the-proofing-viewer}

교정 뷰어에서 URL을 보낼 때 나타나는 옵션이 문서 영역에서 URL을 공유할 때 표시되는 옵션과 동일하게 구성됩니다.

이전에는 이러한 옵션이 현재 기능과 동일한 기능을 제공했지만 해당 조직이 문서 목록의 해당 옵션과 일치하지 않았습니다.

자세한 내용은 섹션을 참조하십시오  [URL 공유](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md#sharing-the-url) 기사  [교정 뷰어에서 증명 공유](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md).

## 언어 교정 뷰어의 설명 목록 개선 사항 {#comments-list-enhancements-in-the-proofing-viewer}

이제 주석 목록에서 다음 개선 사항을 사용할 수 있습니다.

* 이제 주석 텍스트 뒤에 밝은 회색 배경이 표시되어 검토자 이름, 검토 시간 및 증명 페이지 번호와 같은 주석 패널에 표시되는 다른 텍스트와 돋보이게 합니다. 이렇게 하면 사용자가 읽고 싶은 유일한 경우에 피드백만 읽는 것이 더 쉽습니다.

   이전에는 [주석] 패널의 전체 배경이 흰색이었습니다.

   자세한 내용은 를 참조하십시오.

* 이제 댓글에 대한 답글이 댓글 아래에 들여써집니다. 따라서 위에 있는 원래 댓글과 응답을 쉽게 구분할 수 있습니다.

   이전에는 댓글 아래의 답글이 댓글에 왼쪽 정렬되었습니다.

   자세한 내용은 의 를 참조하십시오.

## 교정 뷰어에서 비디오 증명 확대/축소 및 이동 {#zoom-and-pan-video-proofs-in-the-proofing-viewer}

이제 교정 뷰어의 배경이 약간 더 가볍습니다. 이러한 변경 사항으로 검은색 배경과 가장자리가 포함된 교정 콘텐츠를 보다 쉽게 구분할 수 있습니다.

이전에는 교정 뷰어 배경이 검은색이었습니다.

## 언어 교정 뷰어에서 주석을 편집할 때 더 나은 가시성 {#better-visibility-when-a-comment-has-been-edited-in-the-proofing-viewer}

이제 검토자가 변경한 주석에 &quot;편집됨&quot; 레이블이 표시됩니다. 이 위로 마우스를 가져가면 검토자의 이름과 변경 날짜가 나타납니다.

## 언어 교정 뷰어의 주석 목록에서 작업과 연관된 주석 수를 봅니다. {#view-the-number-of-comments-associated-with-an-action-in-the-comment-list-in-the-proofing-viewer}

이제 설명 목록에서 필터 아이콘을 클릭하면 각 작업과 연결된 주석의 수를 확인할 수 있습니다.

작업은 조직에 대해 활성화된 경우에만 주석 목록에서 사용할 수 있습니다.

## 교정 뷰어에서 사용자를 증표에 추가할 때 단계 지정 {#specify-the-stage-when-adding-users-to-a-proof-in-the-proofing-viewer}

검토자를 언어 교정 뷰어에 자동 워크플로우가 있는 증명에 추가할 때 검토자가 증명 작업을 수행할 단계를 지정할 수 있습니다. 이것은 교정 뷰어 외부의 증명에 사용자를 추가할 때 사용할 수 있는 기능과 같습니다.

이전에는 제품 교정 뷰어의 증명에 검토자를 추가했을 때 Workfront에서 현재 활성 상태인 스테이지에 검토자를 배치했습니다.

자세한 내용은 [사용자를 추가하여 증명 공유](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md#sharing-with-individual-users) in [교정 뷰어에서 증명 공유](../../../../review-and-approve-work/proofing/reviewing-proofs-within-workfront/review-a-proof/share-a-proof-in-proofing-viewer.md).

## 대화형 또는 정적 웹 사이트 증명에서 관련 웹 사이트 열기 {#opening-the-associated-website-from-an-interactive-or-static-website-proof}

대화형 또는 정적 웹 콘텐츠가 들어 있는 증명을 검토하는 동안 이제 단추를 클릭하여 교정 뷰어에서 새 브라우저 탭에서 연결된 웹 사이트를 쉽게 열 수 있습니다. 이렇게 하면 두 개의 서로 다른 탭을 사용하여 증명에 표시되는 내용을 실제 웹 사이트와 비교할 수 있습니다.

이전에는 URL 위로 마우스를 이동하고 도구 설명을 클릭하여 연결된 웹 사이트를 열 수 있었습니다. 이 접근 방식은 액세스 가능하고 직관적이었습니다.

## 인쇄 요약 개선 사항 {#print-summary-enhancements}

이제 인쇄 요약 페이지에서 다음과 같은 향상된 기능을 사용할 수 있습니다.

* 이제 증명에 모든 페이지를 포함할지, 주석이 포함된 페이지만 포함할지 여부를 지정할 수 있습니다.

   이전에는 주석이 포함된 페이지만 포함할 수 있었습니다.

* 증명에 단계가 있는 경우 이제 증명의 단계 순서 및 각 단계에 대해 수행된 결정을 보여주는 다이어그램을 포함할 수 있습니다.

   이전에는 이 다이어그램을 사용할 수 없었습니다.

* 이제 여러 마크업이 한 댓글과 연결되어 있으면 해당 주석의 숫자가 [인쇄 요약] 페이지의 각 마크업에 나타납니다.

   이전에는 주석 번호가 주석과 연결된 첫 번째 마크업에만 표시되었습니다. 숫자 없이 표시되는 주석에 대한 다른 마크업입니다.

* 증명이 웹 캡처인 경우 주석이 들어 있는 모든 페이지에는 웹 페이지가 캡처된 해상도가 표시됩니다. 예를 들어, 증명에 모바일 장치용으로 캡처된 페이지와 랩톱 및 데스크탑 컴퓨터용으로 캡처된 동일한 페이지가 포함되어 있는 경우에 유용합니다.
* 이제 인쇄 요약 페이지에서 작업, 작성자 또는 미확인 상태별로 주석을 필터링하면 인쇄 증명뿐만 아니라 PDF 또는 Excel 파일로 내보내는 증명에 필터가 적용됩니다.

자세한 내용은 [Adobe Workfront 내에서 증명 요약 인쇄](../../../../review-and-approve-work/proofing/managing-proofs-within-workfront/print-proof-summary-in-wf.md).
