# Adobe Workfront Planning의 2026년 2분기 릴리스 활동

이 문서에서는 2026년 2분기 릴리스 동안 Workfront Planning에 대해 릴리스되는 기능에 대해 설명합니다.

<!--keep the sentence below for all future quarterly release pages-->

Adobe Workfront Planning에 대해 릴리스된 모든 기능 목록을 보려면 [Adobe Workfront Planning 릴리스 활동: 문서 색인](/help/quicksilver/product-announcements/product-releases/planning-release-activity/planning-release-activity-article-index.md)을 참조하십시오.

## 레코드의 세부 정보 페이지에 표시되는 필드 구성

>[!NOTE]
>
>미리 보기: 2026년 1월 29일
>프로덕션 빠른 릴리스: 2026년 2월 12일
>모두를 위한 프로덕션: 2026년 4월 16일

테이블 보기에 표시된 필드를 기반으로 세부 사항 미리 보기 및 페이지 영역에서 레코드 필드를 표시하거나 숨길 수 있는 설정이 도입되었습니다.

레코드의 세부 정보 영역에서 테이블 보기에 숨겨진 필드를 포함할지 또는 제외할지 선택할 수 있습니다. 이 설정을 끄면 세부 정보 영역에 숨겨진 필드가 표시되지 않습니다.

자세한 내용은 [레코드 페이지 레이아웃 관리](/help/quicksilver/planning/records/manage-the-record-page.md)를 참조하십시오.

## 원래 요청에 대한 새 Planning 연결 필드

>[!NOTE]
>
>미리 보기: 2026년 1월 29일
>프로덕션 빠른 릴리스: 2026년 2월 12일
>모두를 위한 프로덕션: 2026년 4월 16일

이제 &quot;원본 요청&quot; 연결 필드를 레코드 유형에 추가할 수 있습니다. Planning 요청 양식을 실행하여 레코드를 생성하면 원본 요청의 이름이 연결된 원본 요청 필드를 채웁니다.

이 개선 이전에는 요청 영역에서만 레코드를 만든 원래 요청을 보고 액세스할 수 있었습니다.

자세한 내용은 [레코드 종류 연결](/help/quicksilver/planning/architecture/connect-record-types.md)을 참조하세요.

## Workfront Planning에서 요청에 대한 승인 규칙 만들기

>[!NOTE]
>
>미리 보기: 2026년 1월 29일
>프로덕션 빠른 릴리스: 2026년 2월 12일
>모두를 위한 프로덕션: 2026년 4월 16일

요청 승인을 보다 동적이고 유연하게 만들기 위해 승인 규칙을 만드는 기능이 추가되었습니다. 이러한 규칙을 사용하면 요청의 필드 값에 따라 요청을 다른 승인자에게 라우팅할 수 있습니다.

예를 들어 요청 양식에 &quot;캠페인 유형&quot; 필드가 있는 경우, 필드에 &quot;디지털&quot; 값이 있을 경우 한 사람에게, &quot;인쇄&quot; 값이 있을 경우 다른 사람에게 요청을 보내는 규칙을 만들 수 있습니다.

승인 규칙은 요청 양식에 생성됩니다.

자세한 내용 및 지침은 Adobe Workfront Planning의 요청 양식에 승인 추가 문서에서 [요청 양식에 승인 규칙 추가](/help/quicksilver/planning/requests/add-approval-to-request-form.md#add-approval-rules-to-a-request-form)를 참조하십시오.

## Workfront Planning의 사용자 자동 완성 필드에 이메일 표시

>[!NOTE]
>
>미리 보기: 2026년 1월 22일
>프로덕션 빠른 릴리스: 2026년 2월 12일
>모두를 위한 프로덕션: 2026년 4월 16일

이제 다음 영역에 사용자를 추가하면 사용자의 이메일이 표시됩니다.

* 사용자 유형 계획 필드
* Planning 객체를 공유할 때 공유 상자에서

이 개선 이전에는 이러한 영역에 사용자를 추가할 때 사용자 이름 외에 사용자의 기본 역할만 표시되었습니다.
자세한 내용은 다음 문서를 참조하십시오.

* [필드 만들기](/help/quicksilver/planning/fields/create-fields.md)
* [작업 공간 공유](/help/quicksilver/planning/access/share-workspaces.md)
