### 릴리스 정보로 최신 정보 유지

제공된 기능에 대한 릴리스 노트는 해당 기능이 [!UICONTROL 샌드박스 미리 보기] 환경. 이렇게 하면 프로덕션 환경에 포함되기 전에 해당 기능에 대해 알고 탐색할 수 있습니다.

각 분기별 릴리스에는 각 기능에 대한 간단한 설명과 데모 비디오뿐만 아니라 추가 정보를 포함하는 심층적인 참고에 대한 링크가 포함된 분기별 릴리스 개요 페이지가 있습니다. 이러한 심도 있는 메모는 미리 볼 릴리스가 발생할 때 주별로 구성한 다음 실제 프로덕션 릴리스 날짜에 가까운 영역별로 재구성됩니다.

릴리스 노트에서 사용할 수 있는 데모 비디오를 시청하는 것이 좋습니다. 이 기능은 특히 작동 중인 기능을 보거나 시간이 부족한 경우 기능 컨텍스트를 빠르게 살펴보는 데 유용합니다.

분기별 릴리스 개요 링크에는 다음을 참조하십시오. [제품 릴리스](product-releases.md).

### 보기 [!DNL Workfront] 공지 센터

>[!IMPORTANT]
>
>알림 센터 메시지 [!DNL Workfront] 다음에만 사용할 수 있습니다. [!DNL Workfront] 관리자

로서의 [!DNL Workfront] 관리자인 경우 알림 센터 메시지를 받게 됩니다 [!DNL Workfront] 릴리스의 다양한 측면에 대해 알아보십시오. 다음 중 하나가 발생하면 알림 센터 메시지가 표시됩니다.

* 프로덕션 환경에서 새로운 기능을 사용할 수 있습니다
* 미리 보기에 릴리스된 기능은 예약된 분기별 릴리스 전에 프로덕션에 릴리스됩니다.
* 모든 새로운 기능은 예약된 분기별 릴리스 전에 미리 보기 환경에서 사용할 수 있습니다.
* 릴리스 웨비나는 등록할 수 있도록 열려 있습니다

>[!TIP]
>
>선택 취소하여 새 릴리스에 대한 정보를 수신하는지 확인하십시오. &quot;[!UICONTROL 새로운 릴리스]&quot; [!UICONTROL 공지 센터] 설정.
>

### 탐색 [!UICONTROL 샌드박스 미리 보기] 환경

다음 [!UICONTROL 샌드박스 미리 보기] 환경이라고도 합니다 [!UICONTROL 미리 보기] 환경 은 조직의 사용자가 프로덕션 환경에서 별도의 데이터 사본을 안전하게 테스트하고 사용할 수 있는 환경으로 사용됩니다. [!UICONTROL Workfront] 는 프로덕션 환경에 릴리스되기 전에 미리 보기 환경에서 새로운 기능과 기능을 릴리스합니다. 사용자가 제공하는 일상적인 워크플로우에 릴리스하기 전에 이러한 기능을 숙지할 수 있습니다. 그런 다음 사용자가 새로운 기능을 활용하는 데 어려움이 있다면 보다 손쉽게 새로운 기능을 활용할 수 있습니다.

>[!TIP]
>
>미리 보기 환경을 사용하여 프로덕션 환경에 릴리스하기 전에 기능에 대해 테스트하고 학습하는 것이 좋습니다.

기능이 미리 보기 샌드박스 환경에 릴리스되면 [!DNL Workfront] 새 기능을 포함하도록 설명서가 업데이트되었습니다. 해당 문서를 검색하거나 릴리스 노트에서 설명서 링크를 따를 수 있습니다.

에 대한 자세한 내용은 [!UICONTROL 샌드박스 미리 보기] 환경: [다음 [!DNL Adobe Workfront] [!UICONTROL 샌드박스 미리 보기] 환경](../../administration-and-setup/set-up-workfront/workfront-testing-environments/wf-preview-sandbox-environment.md).

### 사용자에게 릴리스 정보 전달

릴리스 정보를 사용자에게 전달할 준비를 할 때 조직에서 해당 정보를 전달하는 가장 효과적인 방법을 고려하십시오. 모든 사용자에게 연락할 수 있도록 다양한 통신 방법을 사용하는 것이 좋습니다. 다음과 같은 통신 채널을 고려할 수 있습니다.

* 내부 뉴스레터
* 이메일
* 알림 센터 메시지(에서 전달됨) [!DNL Workfront] 또는 사용자가 생성)
* 다음과 같은 공동 작업 도구에 대한 공지 [!DNL Slack] 또는 [!DNL Microsoft Teams]
* 내부 웨비나 또는 교육

다음 중 균형을 선택할 수 있습니다 [!DNL Workfront] 조직의 요구 사항과 워크플로우에 더욱 적합한 정보를 제공하는 리소스(예: 설명서, 릴리스 노트 및 교육)를 참조하십시오. 사용자에게 새로운 기능을 보여줌으로써 작업을 향상시키거나 시간과 에너지를 절약하는 것이 사용자가 변경 사항을 수용하는 데 도움이 되지만, 이미 사용 가능한 데모나 설명서를 사용하여 시간을 절약할 수 있습니다 [!DNL Workfront].

시간이 지나면 조직에 가장 효과적인 커뮤니케이션을 인식하게 됩니다. 다음 릴리스에 대한 프로세스를 문서화하고, 성공을 평가하고, 커뮤니케이션을 구체화하는 것이 좋습니다.

### 레이아웃 템플릿을 사용하여 기능 가용성을 제어하는 것이 좋습니다

릴리스에 대한 정보를 수집할 때 사용자가 새 기능을 사용할 준비가 되지 않았다고 결정할 수 있습니다. 이 경우 레이아웃 템플릿을 사용하여 사용자가 보는 내용을 제한할 수 있으므로 새 기능이 표시되지 않고 사용할 수 없습니다. 그런 다음 사용자에게 새 기능에 대한 충분한 통신 및 교육이 제공되면 레이아웃 템플릿을 업데이트하여 기능을 사용할 수 있도록 할 수 있습니다.

레이아웃 템플릿에 대한 자세한 내용은 [레이아웃 템플릿 만들기 및 관리](../../administration-and-setup/customize-workfront/use-layout-templates/create-and-manage-layout-templates.md).

## 다음 작업 [!DNL Workfront] 릴리스 작업을 추적하는 프로젝트

조직이 새로운 릴리스를 사용할 준비가 되었는지 확인하기 위해 프로젝트를 만드는 것이 좋습니다. 여기에서 각 워크플로우 또는 [!DNL Workfront] 미리 보기 환경에서 테스트할 수 있습니다. 워크플로우에 집중하면 기존 워크플로우를 변경할 수 있는 새로운 기능과 영향을 받지 않는 워크플로우를 파악할 수 있습니다. 다음과 같은 작업이 있을 수 있습니다.

* 프로젝트 만들기 테스트
* 증명 승인 테스트
* 에 대한 액세스 수준 테스트 [!UICONTROL 근로자] 및 [!UICONTROL 플래너]
* 모바일 앱 테스트

또한 테스트 외에 릴리스 책임을 추적하는 것이 도움이 될 수 있습니다. 이 프로젝트에 이러한 작업을 추가하거나 새 작업을 만들 수 있습니다. 여기서는 다음과 같은 작업을 추가할 수 있습니다.

* 릴리스 웨비나에 참석
* 데모 비디오 보기

이 프로젝트에서 템플릿을 만들면 각 새로운 릴리스를 준비할 때 일관성과 철저성을 높일 수 있습니다.
